# 목적
핀볼게임에 ai 기술을 도입하기 위해 어떻게 할것인가?

## 간단한 목적
핀볼에 필살기 2개를 만든다.
필살기는 공이 커지고, 작아진다.
사용자의 성향에 따라 필살기가 달라진다.
사용자가 안전주의라면 큰공 필살기가 나오고
사용자가 공격주의라면 작은공 필살기가 나온다.

## 시나리오
1. 사용자가 ai 와 게임을 시작한다.
2. 사용자의 플레이 성향 데이터를 일정시간 수집한다.
- 어떤 데이터를? 언제? 어떻게? 어느정도?
- 사용자 핀볼 플랫퍼 주변에 공을 감지하는 trigger 영역을 만든다.
- 공이 들어오면 공 위치, 속도, 플립퍼 왼쪽,오른쪽 on off 유무를 수집한다.
    -> 숫자 데이터를 전달하면 LLM 이 잘 작동할까?
    -> GEMINI 추천: 숫자 데이터보단 직접 패닉상태인지 코드로 짜라는데,,, 이거 구현 어려울거 같아.
- 초당 10회, 30초 수집


3. 해당 데이터를 ai 에게 분석을 맞겨 안전주의, 공격주의를 분간한다.
- 어떻게 분석을 맞겨? 안전주의, 공격주의는 뭐지?
    -> 데이터를 Json 으로 묶어 LLM 에 넣는다. 이때 각 데이터가 어떤 데이터인지 설명한다.
    공의 위치는 무엇인지, 공의 속도는 무엇인지, 플립퍼는 어디에 있는지, 등등...
    해당 데이터로 무엇을 판단해야하는지. 공격주의, 안전주의
    안전주의란 빠른 승리보다 생존을 원하는것, 초보자의 행동을 보일때. ex) 공이 닿기전에 플립퍼를 연타함. 위험존에 공을 머무르게 하려 하지 않음
    공격주의란 생존의 위험을 감수하고 빠르게 승리하려는것, 고수의 행동을 보일때. ex) 공이 플립퍼에 닿아도 플핍퍼 키를 누르지 않음. 위험존에 공이 오래 머무름.

4. 데이터 분석중에는 필살기를 로딩표시 하여 사용할 수 없게 한다.
- 어떻게 로딩표시를 할꺼지? 필살기 준비를 어떻게 알지?
    -> 스킬 아이콘을 만들고 로딩을 만든다. 끝날때 해당 스킬 아이콘으로 바뀐다.
    스킬 아이콘은 큰 공을 나타내고 작은 공을 나타낼 수 있게 해야한다.

5. 데이터 분석이 끝나 프론트에 보내 필살기를 사용할수 있게 하여 분석이 끝남을 알린다.
- 데이터 분석이 끝남을 어떻게 알지? 데이터 결과를 어떻게 받지? 결과를 어떻게 해석하지? 어떻게 프론트에 알려야지?
    -> 30초 데이터를 수집 후 api 에게 requset 를 보낸다.
    그 후 10 초 타이머를 시작한다. 10초를 기다려 응답을 받으면 해당 결과를 필살기 아이콘에 나타낸다. 만약 오류와 같이 응답이 없거나 10초가 지나면 console 에 표시하고 랜덤한 필살기를 주어야 한다.
    데이터 결과는 respones_schema 를 주어 정해진 답만 답하도록 한다.
    분석 결과를 파싱해 어떤 데이터가 왔는지 프론트에 답해준다. (해당 답변 데이터는 미리 협의 되어있어야함)
    프론트는 해당 데이터를 받고, 10초가 안지났으면 10초 후에 필살기 로딩을 끝내고, 10 초가 지났을때라면 해당 답변을 무시한다. (이미 필살기가 정해저 있음)



### 해야할거
### 핀볼
- 필살기 아이콘
 -> 핀볼 왼쪽 아래 빈 공간에 필살기 표시
 해당 아이콘은 render 이 아닌 react 컴포넌트로 표시
 아이콘은 BIG 과 small 텍스트 (또는 파란 큰 공, 작은 빨간공)

- 필살기 로딩
 -> mui 로딩으로
 게임시작시 계속 로딩 표시로.

 1시간

- 필살기 (큰공, 작은공)
 -> 큰 공은 파란색. 공 반지름을 2배 늘린다. 질량을 +0.5 한다.
 작은공은 빨간색. 공 반지름을 0.3배 줄인다. 질량을 -0.5 한다.
 필살기 키 a 버튼을 누르면 원래 공을 지우고 해당 필살기 공으로 교체한다.
 해당 필살기 공은 5초가 지나면 지우고 원래 공으로 교체한다.
body 2개 생성. 버튼 누를때 해당 위치에 공 교체 및 물리량 복사 (질량, 크기 제외)
5 초 타이머 후 원래 공 교체

1~2시간

- 데이터 분석 수집 로직
 -> 게임 시작시 30초동안 데이터 수집.
 플립퍼 구역 trigger body 생성 및 색 입히기
 해당 trigger body 에 공이 닿을때 현재 시간, 공 위치, 공 벡터, 플립퍼 버튼 누름 에 대해 데이터를 수집한다. trigger body 에 닿을때만 수집하고 1초에 10번 수집한다. 
 30초가 끝나면 해당 데이터를 배열로 api 에 전송한후 응답을 10초동안 기다린다.
 만약 10초 안에 응답이 오지 않으면 무작위로 필살기를 준다. 
 응답을 받으면 해당 응답을 해석해서 해당 필살기를 대기 시킨후 응답 기다리는 타이머가 끝나면 필살기 로딩을 풀고 해당 필살기를 활성화 시킨다.

2시간


### api
- 데이터 받기 api 만들기
 -> /pinball_ai/playstyle
 해당 데이터를 받으면 gemini 1회용 클라이언트 생성(모델은 동일하게)
 해당 데이터를 우선 프롬프트 없이 나오는 값을 response 에 반환.

1 시간

- 시스템 프롬프트 및 response 스키마 작성
 1. 너는 무엇인지
 2. 해야할 작업 설명 (공격적, 수비적 설명)
 3. 들어온 데이터 설명
 4. generation_config 의 "response_schema" 에 대해 설명. -> gemini flase 2.5 에서 사용가능
 5. response_schema 작성.

2시간 시스템프롬프트/response 스키마. -> 스키마 먼저 시스템 프롬프트 그 후.


api 구조
/pinball_ai/playstyle
POST
[{시간, 공 위치, 공 벡터, 플립퍼 왼쪽 버튼 누름, 플립퍼 오른쪽 버튼 누름}, ...]
Resqonse
[{"성공":true, playstyle: attack, defence}]


스키마 예시
```python
import google.generativeai as genai
import typing_extensions as typing
from typing import Literal

# 1. API 키 설정
genai.configure(api_key="YOUR_GEMINI_API_KEY")

# 2. 허용된 값만 가질 수 있도록 스키마 정의
# Literal["A", "B"]는 값의 범위를 'A'와 'B'로만 제한합니다.
class AnalysisResponse(typing.TypedDict):
    user_state: Literal["A", "B"]

# 3. 모델 설정 (JSON 모드 및 스키마 적용)
model = genai.GenerativeModel(
    model_name="gemini-1.5-flash",
    generation_config={
        "response_mime_type": "application/json",
        "response_schema": AnalysisResponse,
    }
)

# 4. 분석할 텍스트 및 프롬프트
# 프롬프트에는 A와 B의 기준만 명확히 적어주면 됩니다.
prompt = """
다음 대화 내용을 분석해서 사용자의 태도를 분류해.
상대가 공격적이면 "A", 방어적이면 "B"라고 응답해.

대화 내용: "왜 자꾸 나한테 그런 식으로 말하는 거야? 짜증 나게 하지 마."
"""

# 5. 실행
response = model.generate_content(prompt)

# 결과 출력
import json
result = json.loads(response.text)
print(f"분석 결과: {result}") # {'user_state': 'A'}
```
