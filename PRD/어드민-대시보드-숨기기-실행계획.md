# 어드민 대시보드 숨기기 실행계획

## 요구사항 요약

**요구사항**: 어드민 페이지 사이드바에서 대시보드 항목을 제거하고, `/admin` 접속 시 회원관리 페이지를 보여준다.

**목적**: 대시보드 기능을 비활성화하여 어드민 진입점을 회원관리 페이지로 통일한다.

## 현재상태 분석

- `AdminSidebar.jsx`: `menuItems` 배열에 대시보드, 회원관리, 통계 3개 항목이 존재
- `App.jsx`: `/admin` 경로가 `AdminPage`(대시보드 레이아웃)로 연결됨
- `AdminPage.jsx`: `AdminMain` 콘텐츠를 렌더링하는 대시보드 전용 컴포넌트
- `AdminMain.jsx`: 대시보드 전용 메인 콘텐츠 컴포넌트 (현재 사용됨)

## 구현 방법

- `AdminSidebar.jsx`의 `menuItems`에서 대시보드 항목을 삭제한다.
- `App.jsx`에서 `/admin` 라우트를 `AdminPage` 대신 `AdminUserPage`로 변경한다.
- `AdminPage`와 `AdminMain` 컴포넌트는 코드만 수정하며 파일은 삭제하지 않는다.

## 구현 단계

### 1. AdminSidebar.jsx - 대시보드 메뉴 항목 제거

```jsx
const menuItems = [
  { text: '회원관리', icon: <People />, path: '/admin' },
  { text: '통계', icon: <BarChart />, path: '/admin/statistics' },
];
```
- **무엇을 하는가**: 사이드바에서 대시보드 항목을 제거하고 회원관리와 통계만 표시
- `DashboardIcon` import도 함께 제거한다.
- `/admin` 경로를 회원관리에 할당하여 진입점과 메뉴 경로를 일치시킨다.

### 2. App.jsx - /admin 라우트 변경

```jsx
import { AdminUserPage, AdminStatisticsPage } from './pages/admin';

// 변경 전
<Route path="/admin" element={<AdminPage/>} />

// 변경 후
<Route path="/admin" element={<AdminUserPage/>} />
```
- **무엇을 하는가**: `/admin` 접속 시 대시보드 대신 회원관리 페이지를 렌더링
- `AdminPage` import를 제거한다.

## 수정/생성할 파일 목록

| 파일 경로 | 작업 유형 | 변경 내용 |
|-----------|-----------|-----------|
| `frontend/src/pages/admin/AdminSidebar.jsx` | 수정 | menuItems에서 대시보드 항목 및 DashboardIcon import 제거 |
| `frontend/src/App.jsx` | 수정 | `/admin` 라우트를 AdminUserPage로 변경, AdminPage import 제거 |

## 완료 체크리스트

- [ ] 사이드바에 대시보드 메뉴 항목이 보이지 않는다.
- [ ] `/admin` 접속 시 회원관리 페이지가 표시된다.
- [ ] 사이드바의 회원관리 항목이 `/admin` 접속 시 선택 상태로 표시된다.
- [ ] 통계 페이지(`/admin/statistics`)가 정상 동작한다.
- [ ] 브라우저 콘솔에 에러가 없다.
