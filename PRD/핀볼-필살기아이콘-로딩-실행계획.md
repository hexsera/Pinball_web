# 핀볼 필살기 아이콘 및 필살기 로딩 실행계획

## 요구사항 요약

**요구사항**: 핀볼 게임 UI에 필살기 아이콘과 로딩 표시를 추가한다.
- 모서리가 둥근 검은 사각형 + 흰색 테두리 프레임 안에 상태를 표시
- 로딩 중: 프레임 안에 MUI CircularProgress 표시
- 분석 완료: 프레임 안에 큰 공(파란색) 또는 작은 공(빨간색) 표시
- 필살기 발동 로직은 구현하지 않음 — UI/상태만 구현

**목적**: 필살기 AI 분석 진행 상태를 사용자에게 시각적으로 알려주고, 분석 결과에 따른 필살기 유형을 아이콘으로 표시한다.

## 현재상태 분석

- `AIPinball.jsx`: 700×1100 게임 영역 + 100px 상단 UI 바로 구성됨
- 게임 영역(`position: relative`)은 `overflow: hidden`, 내부 요소는 `position: absolute`로 배치 가능
- 필살기 관련 상태, 아이콘, 로딩 컴포넌트 전혀 없음
- MUI가 이미 import되어 있음 (`Box`, `Typography`, `IconButton`)
- `WallOverlay.jsx`가 이미 별도 파일로 분리된 선례가 있음

## 구현 방법

- `SkillIcon`을 별도 파일 `frontend/src/pages/AIPinball/SkillIcon.jsx`로 작성
- 필살기 상태를 `AIPinball.jsx`의 React state로 관리: `skillState` — `'loading' | 'big' | 'small'`
- `SkillIcon`은 게임 영역 내부에 `position: absolute`로 배치 (왼쪽 아래)
- 위치: `bottom: '120px'`, `left: '20px'` — 플리퍼 버튼과 겹치지 않는 위치
- 프레임: `borderRadius: '12px'`, 검은 배경, 흰색 테두리 Box
- 프레임 안 콘텐츠: 로딩 시 CircularProgress, 완료 시 원형 공 + 텍스트 레이블

## 구현 단계

### 1. SkillIcon.jsx 파일 생성

```jsx
// frontend/src/pages/AIPinball/SkillIcon.jsx
import { Box, Typography } from '@mui/material';
import CircularProgress from '@mui/material/CircularProgress';

function SkillIcon({ skillState }) {
  return (
    <Box sx={{
      width: '80px',
      height: '80px',
      borderRadius: '12px',
      backgroundColor: '#000000cc',
      border: '2px solid #ffffff',
      display: 'flex',
      flexDirection: 'column',
      alignItems: 'center',
      justifyContent: 'center',
      gap: '6px',
    }}>
      {skillState === 'loading' && (
        <>
          <CircularProgress size={32} sx={{ color: '#ffffffaa' }} />
          <Typography sx={{ color: '#ffffff88', fontSize: '9px', letterSpacing: '0.5px' }}>
            LOADING
          </Typography>
        </>
      )}
      {skillState === 'big' && (
        <>
          <Box sx={{
            width: '38px', height: '38px', borderRadius: '50%',
            backgroundColor: '#2196f3',
            boxShadow: '0 0 10px #2196f3',
            border: '2px solid #64b5f6',
          }} />
          <Typography sx={{ color: '#64b5f6', fontSize: '9px', fontWeight: 'bold', letterSpacing: '0.5px' }}>
            BIG
          </Typography>
        </>
      )}
      {skillState === 'small' && (
        <>
          <Box sx={{
            width: '20px', height: '20px', borderRadius: '50%',
            backgroundColor: '#e94560',
            boxShadow: '0 0 10px #e94560',
            border: '2px solid #ff6b6b',
          }} />
          <Typography sx={{ color: '#ff6b6b', fontSize: '9px', fontWeight: 'bold', letterSpacing: '0.5px' }}>
            SMALL
          </Typography>
        </>
      )}
    </Box>
  );
}

export default SkillIcon;
```

- 외부 프레임: 80×80px, `borderRadius: 12px` 둥근 검은 사각형, 흰색 2px 테두리
- 로딩: 프레임 안에 CircularProgress(32px) + "LOADING" 텍스트
- BIG: 프레임 안에 파란 원(38px) + "BIG" 텍스트
- SMALL: 프레임 안에 빨간 원(20px) + "SMALL" 텍스트 — 크기 차이로 big/small 직관적으로 구분

### 2. AIPinball.jsx에 import 및 skillState 상태 추가

```javascript
// import 추가 (AIPinball.jsx 상단)
import SkillIcon from './SkillIcon';

// useState 목록에 추가 (기존 const [isMuted, ...] 아래)
const [skillState, setSkillState] = useState('loading');
```

- `SkillIcon`은 같은 디렉토리이므로 `'./SkillIcon'`으로 import
- 초기값 `'loading'`: 게임 시작 전부터 로딩 표시

### 3. 게임 영역 내부 왼쪽 아래에 SkillIcon 배치

```jsx
{/* 게임 영역 Box 내부, <canvas ref={sceneRef} /> 아래에 추가 */}
<Box sx={{
  position: 'absolute',
  bottom: '120px',
  left: '20px',
  zIndex: 5,
}}>
  <SkillIcon skillState={skillState} />
</Box>
```

- 게임 영역 Box는 이미 `position: 'relative'`이므로 `absolute` 자식이 그 안에 고정됨
- `bottom: '120px'`: 모바일 플리퍼 버튼(bottom: 40px + 100px) 위로 여백 확보
- `zIndex: 5`: 캔버스(z-index: 2) 위, 오버레이(z-index: 10) 아래
- 삽입 위치: `AIPinball.jsx` 1138라인 `<div ref={sceneRef} />` 바로 아래

### 4. (선택) 개발/테스트용 임시 키 연동

```javascript
// handleKeyDown 함수 내부 기존 'n' 키 처리 아래에 추가
if (event.key === 'b' || event.key === 'B') {
  setSkillState('big');
}
if (event.key === 's' || event.key === 'S') {
  setSkillState('small');
}
if (event.key === 'l' || event.key === 'L') {
  setSkillState('loading');
}
```

- B 키: BIG 아이콘으로 전환 (개발 확인용)
- S 키: SMALL 아이콘으로 전환 (개발 확인용)
- L 키: 로딩 상태로 복귀 (개발 확인용)
- 이후 AI API 연동 시 해당 키 코드 제거 후 API 응답으로 대체
