# 홈(랜딩 페이지) 생성 실행계획

## 요구사항 요약

**요구사항**: `/` 경로에 홈(랜딩 페이지) 컴포넌트(`HomePage.jsx`)를 새로 생성하고, 기존 Dashboard는 `/dashboard` 경로로 이동한다.

**목적**: 사이트 첫 진입 시 게임 소개, CTA 버튼, 이번 달 랭킹 테이블을 보여주는 랜딩 페이지를 제공하여 신규 방문자 유입 및 게임 참여를 유도한다.

---

## 현재상태 분석

- `App.jsx`에서 `/` → `Dashboard` 컴포넌트가 직접 연결되어 있음
- 랜딩 페이지 역할의 컴포넌트가 없음
- 프론트엔드는 MUI + React Router 기반으로 구성되어 있음

---

## 구현 방법

- **프론트엔드**: MUI `Box`, `Grid`, `Table`, `AppBar`, `Button` 컴포넌트로 레이아웃 구성
- **컬러**: 핀볼 테마 인디고(`#4F46E5`) 포인트 컬러, 다크 네이비(`#0F172A`) 배경 사용
- **랭킹 데이터**: 더미 데이터를 `useState` 초기값으로 하드코딩하여 상위 5명 표시
- **반응형**: MUI Grid `xs=12 / md=6` 으로 모바일 단일 컬럼 / 데스크톱 2단 분할

---

## 구현 단계

### 1. HomePage 컴포넌트 파일 생성

```jsx
// frontend/src/pages/HomePage/HomePage.jsx
import { useState } from 'react';
import { useNavigate } from 'react-router-dom';
import {
  Box, AppBar, Toolbar, Typography, Button, Grid,
  Table, TableBody, TableCell, TableContainer,
  TableHead, TableRow, Paper
} from '@mui/material';

const COLORS = {
  bg: '#0F172A',
  card: '#1E293B',
  border: '#334155',
  text: '#F1F5F9',
  subText: '#94A3B8',
  primary: '#4F46E5',
};
```
- **무엇을 하는가**: 홈페이지 전용 컴포넌트 파일과 색상 상수를 정의
- 다크 네이비 배경(`#0F172A`)과 인디고 포인트(`#4F46E5`)로 핀볼 테마 구성
- `COLORS` 객체로 색상을 한 곳에서 관리하여 유지보수를 쉽게 함

---

### 2. 더미 랭킹 데이터 상태 정의

```jsx
const [ranking] = useState([
  { rank: 1, nickname: 'alpha', score: 1234500 },
  { rank: 2, nickname: 'ninja', score: 987300 },
  { rank: 3, nickname: 'hexfan', score: 765000 },
  { rank: 4, nickname: 'king', score: 543200 },
  { rank: 5, nickname: '홍길동', score: 321000 },
]);
```
- **무엇을 하는가**: 랭킹 테이블에 표시할 더미 데이터를 컴포넌트 상태로 정의
- `useState` 초기값으로 하드코딩하여 API 없이도 화면이 동작하게 처리
- setter 함수 없이 선언하여 이 데이터는 변경되지 않음을 명시

---

### 3. 네비게이션 바 구현

```jsx
<AppBar position="static" sx={{ backgroundColor: COLORS.card, borderBottom: `1px solid ${COLORS.border}` }}>
  <Toolbar sx={{ justifyContent: 'space-between' }}>
    <Typography variant="h6" sx={{ color: COLORS.primary, fontWeight: 'bold' }}>
      🎯 HEXSERA PINBALL
    </Typography>
    <Box>
      <Button onClick={() => navigate('/dashboard')} sx={{ color: COLORS.text }}>
        게임하기
      </Button>
      <Button onClick={() => navigate('/login')} variant="outlined"
        sx={{ color: COLORS.primary, borderColor: COLORS.primary, ml: 1 }}>
        로그인
      </Button>
      <Button onClick={() => navigate('/Register')} variant="contained"
        sx={{ backgroundColor: COLORS.primary, ml: 1 }}>
        회원가입
      </Button>
    </Box>
  </Toolbar>
</AppBar>
```
- **무엇을 하는가**: 상단 고정 네비게이션 바에 로고, 게임하기 링크, 로그인·회원가입 버튼 배치
- `justifyContent: 'space-between'`으로 로고(좌)와 버튼(우) 양쪽 배치
- 로그인: outlined 버튼, 회원가입: contained(채운) 버튼으로 시각적 우선순위 구분

---

### 4. Hero 섹션 (좌측: CTA, 우측: 랭킹 테이블) 구현

```jsx
<Grid container spacing={4} sx={{ mt: 4, px: { xs: 2, md: 8 } }}>
  {/* 좌측: 타이틀 + CTA */}
  <Grid item xs={12} md={6}>
    <Box sx={{ display: 'flex', flexDirection: 'column', gap: 3 }}>
      <Typography variant="h3" sx={{ color: COLORS.text, fontWeight: 'bold' }}>
        세상에서 가장 짜릿한<br/>핀볼 게임을 경험하세요.
      </Typography>
      <Button variant="contained" size="large"
        onClick={() => navigate('/dashboard')}
        sx={{ backgroundColor: COLORS.primary, width: 'fit-content', px: 4, py: 1.5 }}>
        ▶ 지금 바로 플레이하기
      </Button>
    </Box>
  </Grid>

  {/* 우측: 이번 달 랭킹 */}
  <Grid item xs={12} md={6}>
    <Paper sx={{ backgroundColor: COLORS.card, border: `1px solid ${COLORS.border}`, p: 3, borderRadius: 2 }}>
      <Typography variant="h6" sx={{ color: COLORS.text, mb: 2 }}>🏆 이번 달 랭킹</Typography>
      <TableContainer>
        <Table size="small">
          <TableHead>
            <TableRow>
              <TableCell sx={{ color: COLORS.subText }}>#</TableCell>
              <TableCell sx={{ color: COLORS.subText }}>닉네임</TableCell>
              <TableCell align="right" sx={{ color: COLORS.subText }}>점수</TableCell>
            </TableRow>
          </TableHead>
          <TableBody>
            {ranking.map((row) => (
              <TableRow key={row.rank}>
                <TableCell sx={{ color: COLORS.text }}>
                  {row.rank === 1 ? '🥇' : row.rank === 2 ? '🥈' : row.rank === 3 ? '🥉' : row.rank}
                </TableCell>
                <TableCell sx={{ color: COLORS.text }}>{row.nickname}</TableCell>
                <TableCell align="right" sx={{ color: COLORS.primary, fontWeight: 'bold' }}>
                  {row.score.toLocaleString()}
                </TableCell>
              </TableRow>
            ))}
          </TableBody>
        </Table>
      </TableContainer>
    </Paper>
  </Grid>
</Grid>
```
- **무엇을 하는가**: PRD 설계대로 좌측 CTA + 우측 랭킹 테이블을 2단으로 배치
- `Grid xs=12 md=6`으로 모바일 단일 컬럼, 데스크톱 2단 레이아웃 자동 전환
- `row.rank === 1 ? '🥇' : ...` 삼항 연산자로 1~3위 메달 아이콘 표시
- `toLocaleString()`으로 점수에 1,000 단위 쉼표 자동 적용

---

### 5. 푸터 구현

```jsx
<Box component="footer" sx={{ textAlign: 'center', py: 3, mt: 'auto', color: COLORS.subText }}>
  <Typography variant="body2">© 2026 HEXSERA</Typography>
</Box>
```
- **무엇을 하는가**: 페이지 하단에 저작권 텍스트를 표시하는 푸터 영역 구현
- `mt: 'auto'`로 콘텐츠가 짧아도 항상 하단에 위치하도록 처리 (부모에 `minHeight: '100vh'`, `display: 'flex'`, `flexDirection: 'column'` 필요)

---

### 6. index.js 배럴 파일 생성

```js
// frontend/src/pages/HomePage/index.js
export { default } from './HomePage';
```
- **무엇을 하는가**: `import HomePage from '../HomePage'` 형태의 단축 import를 가능하게 하는 배럴 파일
- 다른 pages 폴더(`Login/index.js`, `Dashboard/index.js`)와 동일한 패턴 유지

---

### 7. App.jsx 라우트 수정

```jsx
import HomePage from './pages/HomePage';
// ...기존 import 유지...

<Routes>
  <Route path="/" element={<HomePage />} />           {/* 변경: Dashboard → HomePage */}
  <Route path="/dashboard" element={<Dashboard />} /> {/* 추가: 기존 Dashboard */}
  <Route path="/login" element={<Login />} />
  <Route path="/Register" element={<Register />} />
  <Route path="/Pinball_test" element={<Pinball />} />
  <Route path="/admin" element={<AdminPage />} />
  <Route path="/admin/users" element={<AdminUserPage />} />
  <Route path="/admin/statistics" element={<AdminStatisticsPage />} />
</Routes>
```
- **무엇을 하는가**: `/` 경로를 `HomePage`로 변경하고, 기존 `Dashboard`는 `/dashboard` 경로로 이동
- Dashboard를 수정하지 않고 새 경로로만 연결하여 기존 기능을 그대로 유지
- 로그인 후 `navigate('/')` 코드가 있다면 `/dashboard`로 변경 필요 (Login.jsx 확인)

| 파일 경로 | 작업 유형 | 변경 내용 |
