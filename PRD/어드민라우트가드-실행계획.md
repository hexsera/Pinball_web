# 어드민 라우트 가드 실행계획

## 요구사항 요약

**요구사항**: `/admin`, `/admin/users`, `/admin/statistics` 라우트에 진입 시 `role === 'admin'`인 사용자만 허용하는 프론트 가드 컴포넌트를 도입한다.

**목적**: 일반 user나 비로그인 사용자가 어드민 화면에 접근하지 못하도록 UX 차원에서 차단하고 `/`로 리다이렉트한다. 보안 본체는 백엔드 API 검증이며, 이 가드는 안내 역할만 담당한다.

## 현재상태 분석

- [frontend/src/App.jsx:33-35](../frontend/src/App.jsx#L33-L35): 어드민 라우트 3개가 가드 없이 노출. 비로그인·일반 user 모두 진입 가능.
- [frontend/src/store/authStore.js](../frontend/src/store/authStore.js): Access Token은 `useAuthStore`의 `accessToken` 필드에 메모리(Zustand)로 보관. `null`이면 비로그인 상태.
- `jwt-decode` 패키지 미설치. `react-router-dom` v7은 이미 설치됨.
- `frontend/src/components/AdminRoute.jsx` 파일 없음 (신규 생성 필요).
- 기존 테스트는 `frontend/src/test/` 아래 Vitest 기반으로 작성됨.

## 구현 방법

- **Outlet 패턴**: 어드민 라우트 3개를 부모 `<Route element={<AdminRoute />}>`로 묶어 한 번에 보호.
- **JWT payload 디코딩**: `jwt-decode` 라이브러리로 Access Token에서 `role` 필드만 읽는다. 서명 검증은 하지 않는다(백엔드 책임).
- **리다이렉트 정책**: 비로그인·일반 user·디코딩 실패 모두 `/`로 통일 리다이렉트. 알림 없음.

## 구현 단계

### 1. jwt-decode 패키지 설치

```bash
source ~/.nvm/nvm.sh && cd frontend && npm install jwt-decode
```

- `jwt-decode`는 base64url 디코딩·패딩 처리를 내부에서 해결해주는 1KB 가량의 경량 라이브러리다.
- 직접 구현 시 패딩 버그가 발생하기 쉬우므로 라이브러리를 사용한다.

### 2. AdminRoute.jsx 신규 생성

```jsx
// frontend/src/components/AdminRoute.jsx
import { Navigate, Outlet } from 'react-router-dom';
import { jwtDecode } from 'jwt-decode';
import { useAuthStore } from '../store/authStore';

export default function AdminRoute() {
  const accessToken = useAuthStore((s) => s.accessToken);
  if (!accessToken) return <Navigate to="/" replace />;

  try {
    const { role } = jwtDecode(accessToken);
    if (role !== 'admin') return <Navigate to="/" replace />;
  } catch {
    return <Navigate to="/" replace />;
  }

  return <Outlet />;
}
```

- `useAuthStore((s) => s.accessToken)`: 토큰이 변경될 때 컴포넌트가 자동으로 리렌더링된다.
- `accessToken`이 `null`이면 비로그인으로 판정해 즉시 `/`로 보낸다.
- `jwtDecode` 실패(잘못된 토큰 형식)는 `catch`에서 잡아 `/`로 보낸다.
- 모든 조건 통과 시 `<Outlet />`을 반환해 자식 라우트를 렌더링한다.

### 3. App.jsx — 어드민 라우트 Outlet 패턴으로 변경

```jsx
// frontend/src/App.jsx — import 추가
import AdminRoute from './components/AdminRoute';

// 기존 3개 라우트를 아래로 교체
<Route element={<AdminRoute />}>
  <Route path="/admin" element={<AdminUserPage/>} />
  <Route path="/admin/users" element={<AdminUserPage/>} />
  <Route path="/admin/statistics" element={<AdminStatisticsPage/>} />
</Route>
```

- 부모 `<Route>`에 `path`가 없고 `element`만 있으면 Outlet 래퍼로 동작한다.
- 자식 3개 라우트는 기존과 동일하게 유지한다(`/admin`과 `/admin/users` 중복 매핑 정리는 이번 작업 범위 밖).

### 4. AdminRoute.test.jsx 작성

```jsx
// frontend/src/test/AdminRoute.test.jsx
import { render, screen } from '@testing-library/react';
import { MemoryRouter, Routes, Route } from 'react-router-dom';
import { describe, it, expect, vi, beforeEach } from 'vitest';
import AdminRoute from '../components/AdminRoute';
import { useAuthStore } from '../store/authStore';

vi.mock('jwt-decode', () => ({
  jwtDecode: vi.fn(),
}));

import { jwtDecode } from 'jwt-decode';

function setup(token, role) {
  useAuthStore.setState({ accessToken: token });
  if (token && role) jwtDecode.mockReturnValue({ role });
  if (token && !role) jwtDecode.mockImplementation(() => { throw new Error(); });

  return render(
    <MemoryRouter initialEntries={['/admin']}>
      <Routes>
        <Route element={<AdminRoute />}>
          <Route path="/admin" element={<div>Admin Page</div>} />
        </Route>
        <Route path="/" element={<div>Home</div>} />
      </Routes>
    </MemoryRouter>
  );
}

describe('AdminRoute', () => {
  beforeEach(() => vi.clearAllMocks());

  it('토큰 없으면 Home으로 리다이렉트', () => {
    setup(null, null);
    expect(screen.getByText('Home')).toBeInTheDocument();
  });

  it('role=admin이면 어드민 페이지 렌더', () => {
    setup('valid.token.here', 'admin');
    expect(screen.getByText('Admin Page')).toBeInTheDocument();
  });

  it('role=user면 Home으로 리다이렉트', () => {
    setup('valid.token.here', 'user');
    expect(screen.getByText('Home')).toBeInTheDocument();
  });

  it('디코딩 실패하면 Home으로 리다이렉트', () => {
    setup('broken.token', null);
    expect(screen.getByText('Home')).toBeInTheDocument();
  });
});
```

- `MemoryRouter`로 브라우저 없이 라우팅을 시뮬레이션한다.
- `jwtDecode`를 모킹해 실제 JWT 없이 role 분기를 테스트한다.
- `useAuthStore.setState`로 스토어 상태를 직접 주입한다.
