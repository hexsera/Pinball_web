# 공지사항 기능 PRD

## 목표

관리자가 공지사항을 작성하고, 일반 사용자가 공지사항 목록과 상세 내용을 조회할 수 있는 기능을 구현한다.

## 현재상태 분석

- **DB**: `users`, `friendships`, `monthly_scores`, `game_visits` 테이블 존재. 공지사항 테이블 없음.
- **Backend**: FastAPI, SQLAlchemy, Alembic 사용. 라우터는 `/api/v1/` 접두사 사용. `role` 컬럼으로 관리자 구분 가능.
- **Frontend**: React + MUI. 관리자 페이지(`/admin`, `/admin/users`, `/admin/statistics`) 이미 존재. 공지사항 관련 페이지 없음.
- **인증**: localStorage에 `{user_id, email, nickname, role}` 저장. role이 'admin'이면 관리자.

## 실행 방법 및 단계

1. **DB**: `notices` 테이블 추가 (모델 작성 → Alembic 마이그레이션)
2. **Backend Schema**: `NoticeCreate`, `NoticeResponse` Pydantic 스키마 작성
3. **Backend API**: `/api/v1/notices` 라우터 작성 (CRUD 엔드포인트)
4. **Backend 등록**: `main.py`에 notices 라우터 추가
5. **Frontend - 사용자**: 공지사항 목록 페이지(`/notices`) 및 상세 페이지(`/notices/:id`) 구현
6. **Frontend - 관리자**: 어드민 사이드바에 공지사항 메뉴 추가, 작성/수정/삭제 UI 구현 (`/admin/notices`)
7. **Frontend 라우팅**: `App.jsx`에 공지사항 관련 라우트 추가

## 사용 할 기술 및 패키지

| 기술/패키지 | 용도 |
|-------------|------|
| SQLAlchemy | notices 모델 정의 |
| Alembic | DB 마이그레이션 |
| FastAPI | REST API 엔드포인트 |
| Pydantic | 요청/응답 데이터 검증 |
| React + MUI | 공지사항 목록/상세/관리 UI |
| react-router-dom | 공지사항 페이지 라우팅 |

## DB 스키마

```
notices 테이블
- id: Integer, PK
- title: String(255), NOT NULL
- content: Text, NOT NULL
- author_id: Integer, FK(users.id), NOT NULL
- is_pinned: Boolean, default False  (상단 고정 여부)
- created_at: DateTime, server_default=now()
- updated_at: DateTime, server_default=now(), onupdate=now()
```

## API 엔드포인트

| Method | Path | 권한 | 설명 |
|--------|------|------|------|
| GET | `/api/v1/notices` | 모두 | 공지사항 목록 조회 (페이지네이션) |
| GET | `/api/v1/notices/{id}` | 모두 | 공지사항 상세 조회 |
| POST | `/api/v1/notices` | admin | 공지사항 작성 |
| PUT | `/api/v1/notices/{id}` | admin | 공지사항 수정 |
| DELETE | `/api/v1/notices/{id}` | admin | 공지사항 삭제 |

## 테스트 방법

1. 관리자 계정으로 로그인 후 `/admin/notices`에서 공지사항 작성
2. 비로그인 상태에서 `/notices` 접근하여 목록 조회
3. 공지사항 항목 클릭 시 `/notices/:id` 상세 페이지 진입
4. 관리자 계정으로 기존 공지사항 수정 및 삭제
5. 일반 유저 계정으로 POST `/api/v1/notices` 직접 호출 시 403 반환 확인
