# 공지사항 캐시 실행계획

## 요구사항 요약

**요구사항**: 공지사항 목록(`GET /api/v1/notices`)과 상세(`GET /api/v1/notices/{id}`) API에 HTTP 캐시를 적용한다.

**목적**: 공지사항 응답(특히 base64 이미지가 포함된 상세)을 반복 전송하지 않아 서버 트래픽을 줄인다. 동시에 Cache-Control과 ETag의 동작 차이를 직접 체감하며 익힌다.

## 현재상태 분석

- `NoticeListPage`, `NoticeDetailPage` 모두 `useEffect + useState`로 직접 fetch (React Query 없음)
- `noticeService.js`: axios 직접 호출, API 실패 시 IndexedDB fallback
- `notices.py`: `Cache-Control`, `ETag` 헤더 없음 — 브라우저 캐시 불가 상태
- 상세 응답에 base64 이미지가 HTML 안에 포함되어 응답 크기가 큼

## 구현 방법

- **목록**: FastAPI 응답에 `Cache-Control: public, max-age=600` 헤더 추가 → 10분간 브라우저가 요청 자체를 생략
- **상세**: `Cache-Control: public, max-age=3600` + `ETag` 헤더 추가 → 1시간 캐시, 만료 후 내용 불변이면 서버가 304만 응답해 바디 전송 생략
- FastAPI의 `Response` 객체로 헤더를 직접 설정; ETag는 응답 바디의 MD5 해시값 사용

## 구현 단계

### 1. 목록 API에 Cache-Control 헤더 추가

```python
from fastapi import APIRouter, Depends, Response
import json

@router.get("", response_model=NoticeListResult)
def get_notices(skip: int = 0, limit: int = 10, db: Session = Depends(get_db), response: Response = None):
    total = db.query(Notice).count()
    items = db.query(Notice).order_by(Notice.created_at.desc()).offset(skip).limit(limit).all()
    # 목록은 ETag 없이 max-age만 적용 — 글 하나만 바뀌어도 ETag가 달라져 절약 효과 없음
    response.headers["Cache-Control"] = "public, max-age=600"
    return {"items": items, "total": total}
```
- `Response` 객체를 의존성으로 주입받아 헤더를 설정
- `max-age=600`: 10분 동안 브라우저가 서버에 요청을 보내지 않고 캐시를 그대로 사용
- 목록에 ETag를 쓰지 않는 이유: 글 하나만 추가/삭제돼도 ETag가 바뀌어 매번 새 바디를 받아야 해서 이득 없음

### 2. 상세 API에 Cache-Control + ETag 헤더 추가

```python
import hashlib, json
from fastapi import Request
from fastapi.responses import Response as FastAPIResponse

@router.get("/{notice_id}")
def get_notice(notice_id: int, request: Request, db: Session = Depends(get_db)):
    notice = db.query(Notice).filter(Notice.id == notice_id).first()
    if not notice:
        raise HTTPException(status_code=404, detail="Notice not found")

    body = json.dumps({
        "id": notice.id,
        "title": notice.title,
        "content": notice.content,
        "author_id": notice.author_id,
        "created_at": notice.created_at.isoformat(),
        "updated_at": notice.updated_at.isoformat(),
    }, ensure_ascii=False)

    # 응답 바디의 MD5 해시로 ETag 생성 — 내용이 같으면 항상 동일한 값 반환
    etag = f'"{hashlib.md5(body.encode()).hexdigest()}"'

    if request.headers.get("If-None-Match") == etag:
        # 클라이언트가 보낸 ETag와 일치 → 내용 변경 없음, 바디 없이 304 반환
        return FastAPIResponse(status_code=304, headers={"ETag": etag, "Cache-Control": "public, max-age=3600"})

    return FastAPIResponse(
        content=body,
        media_type="application/json",
        headers={
            "Cache-Control": "public, max-age=3600",
            "ETag": etag,
        }
    )
```
- ETag 값은 응답 바디 전체의 MD5 해시 — 내용이 안 바뀌면 항상 동일한 ETag
- `If-None-Match` 헤더: 브라우저가 캐시 만료 후 재요청 시 이전에 받은 ETag를 함께 전송
- 값이 일치하면 서버는 304만 응답해 base64 이미지가 포함된 무거운 바디 전송을 생략
- `response_model` 제거: `FastAPIResponse`를 직접 반환하므로 Pydantic 직렬화를 우회
