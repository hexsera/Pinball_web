# 목표
backend/main.py 에 "/api/friend-requests" 에 이미 구현된 것을 기반으로 RED 를 작성한다.
해당 api 는 요청 id, 요청받는 id 를 제공받아 DB 에 저장한다.

## 상세사항
조건은 다음과 같다.
요청 id -> 요청받는 id 라고 한다면
1. A 유저 -> A 유저 는 안된다.
2. 이미 있는 요청은 중복되면 안된다.
3. A 유저 -> B 유저 가 있을때 B 유저 -> A 유저 요청은 중복됐다고 판단한다.
4. 요청 id 와 요청받는 id 는 user 테이블 id(PK)의 FK 이다. 만약 해당 id 가 없을때 오류처리가 있어야한다.

TDD 스킬을 사용해 RED 계획을 md 로 작성해라. 그리고 기다려라.
