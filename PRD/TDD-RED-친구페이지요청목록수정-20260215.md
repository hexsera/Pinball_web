# TDD RED 계획: 친구페이지 요청목록 및 친구목록 표시 수정

## 구현 목표
FriendPage의 오른쪽 영역(친구 요청 + 현재 친구)의 표시 내용을 PRD 요구사항에 맞게 수정한다.

1. 친구 요청란: receiver_id가 나인 요청만 필터링, requester_nickname 표시, 승인 버튼 초록색
2. 현재 친구란: accepted 친구 목록에서 상대방 닉네임 표시 (양방향 처리)

## RED 테스트 목록

### 친구 요청 섹션
1. `receiver_id가 내 id인 요청만 친구 요청란에 표시된다`: requester_id가 나인 요청(내가 보낸 것)은 표시 안 됨
2. `친구 요청란에 requester_nickname이 표시된다`: "요청 ID: N" 대신 요청자 닉네임 표시
3. `승인 버튼이 초록색(success color)이다`: 승인 버튼의 color prop이 success

### 현재 친구 섹션
4. `receiver_id가 내 id인 경우 requester_nickname을 친구 닉네임으로 표시한다`: 상대가 요청자 → 상대 닉네임은 requester_nickname
5. `requester_id가 내 id인 경우 receiver_nickname을 친구 닉네임으로 표시한다`: 내가 요청자 → 상대 닉네임은 receiver_nickname
