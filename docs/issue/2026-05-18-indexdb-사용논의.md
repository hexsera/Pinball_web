# IndexDB 사용 논의

## 아이디어

공지사항 작성 중 페이지를 벗어나거나 탭을 닫아도 작성 내용이 유지되도록, 작성 세션을 IndexedDB에 임시 저장한다. 다시 작성 페이지에 진입하면 저장된 임시본을 불러와 이어서 작성할 수 있게 한다.

## 일반적인 사용법과의 비교

교과서적인 IndexedDB 사용 사례. 네트워크 없이도, 탭을 닫아도 유지되어야 하는 데이터를 로컬에 보관하는 용도로 IndexedDB가 자주 쓰인다. Notion, Google Docs 등도 같은 방식으로 임시저장(draft)을 구현한다.

## 구현 시 고려사항

- 임시저장 키: 사용자 ID 기준으로 저장 (로그인 유저별 독립 draft)
- 저장 타이밍: 입력할 때마다 저장 (debounce 적용) 또는 일정 주기로 자동저장
- 불러오기: 작성 페이지 진입 시 IndexedDB에 draft가 있으면 복원 여부를 사용자에게 물어봄
- 삭제 타이밍: 글 제출 성공 후 draft 삭제
