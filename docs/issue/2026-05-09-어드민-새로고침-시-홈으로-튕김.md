# 어드민 페이지 새로고침 시 `/`로 튕김

- 발견일: 2026-05-09
- 관련 작업: [docs/history/2026-05-09-a663bc-01-어드민라우트가드-토론.md](../history/2026-05-09-a663bc-01-어드민라우트가드-토론.md) (D-1, D-2, D-3 결정 적용 시 알려진 한계로 분리됨)
- 관련 이슈: [2026-05-07-보호-라우트-가드-미구현.md](2026-05-07-보호-라우트-가드-미구현.md) — 같은 원인의 일반화된 형태

## 증상

`role=admin`으로 로그인한 사용자가 `/admin`, `/admin/users`, `/admin/statistics` 중 한 페이지에서 **브라우저 새로고침**을 누르면, 정상이라면 그대로 어드민 페이지가 다시 보여야 한다. 그러나 현재는 한순간 `/`(홈)로 리다이렉트된 뒤 그 자리에서 머문다. 사용자는 로그인 상태인데도 어드민 화면을 잃는다.

## 재현 절차

1. admin 계정으로 로그인
2. `/admin/users` 또는 `/admin/statistics`로 이동 (정상 표시 확인)
3. 브라우저 새로고침(F5 또는 Ctrl+R)
4. → 화면이 `/`로 바뀜

## 원인

새로고침 직후의 시점에서 다음 순서로 동작한다:

```
1. JS 메모리 초기화 → Zustand의 accessToken 비어 있음
2. 라우트 렌더 시작 → AdminRoute 가드 실행
3. AdminRoute: accessToken === null 이므로 <Navigate to="/" replace /> 결정
4. (한 박자 뒤) App에서 /api/v1/auth/refresh 호출 → Refresh 쿠키로 새 Access 받아 메모리 채움
5. 그러나 이미 라우트는 /로 옮겨진 상태
```

즉 **토큰 복원이 가드 판정보다 늦게 일어나기 때문에**, 가드가 "비로그인"으로 잘못 판정한다. 가드 자체의 결함이 아니라, 가드와 토큰 복원의 타이밍 정합성이 잡혀 있지 않은 것이 원인이다.

이 구조는 어드민에 한정되지 않고 향후 추가될 모든 보호 라우트에 동일하게 영향을 준다. [2026-05-07-보호-라우트-가드-미구현.md](2026-05-07-보호-라우트-가드-미구현.md)에서 언급된 일반 보호 라우트 부재 문제와 같은 근본 원인을 공유한다.

## 영향 범위

- `/admin`, `/admin/users`, `/admin/statistics` — 새로고침 시 매번 발생
- 향후 `<AdminRoute>` 패턴으로 보호되는 모든 페이지에서 동일 증상 예상
- 데이터 노출이나 권한 우회와는 무관 (UX 한정)

## 해결 방향 (후속 세션에서 결정 예정)

가드가 동작하기 전에 토큰 복원 시도가 끝나도록 **로딩 상태**를 도입한다. 골격 후보:

```jsx
// 개념: 인증 부트스트랩이 끝날 때까지 가드 판정을 보류
export default function AdminRoute() {
  const accessToken = useAuthStore((s) => s.accessToken);
  const authReady   = useAuthStore((s) => s.authReady); // refresh 시도 완료 플래그

  if (!authReady) return null; // 또는 로딩 스피너
  if (!accessToken) return <Navigate to="/" replace />;
  ...
}
```

후속 논의 항목:
- `authReady` 플래그를 어디에 둘지 (Zustand 필드 추가 vs 별도 스토어)
- `/auth/refresh` 호출 위치 (App 최상위? 별도 부트스트랩 컴포넌트?)
- 로딩 중 화면 처리 (빈 화면 / 스피너 / 스플래시)
- 어드민 외 일반 보호 라우트(`PrivateRoute`)와의 통합 여부 — 2026-05-07 이슈와 함께 풀지 결정

## 임시 회피책

사용자 입장에서: 새로고침 후 `/admin`을 다시 클릭/입력하면 토큰이 이미 복원된 상태이므로 정상 진입된다.

코드 측 회피책은 도입하지 않는다 — 근본 해결을 후속 세션에서 다룰 예정.
