# 파일 스토리지 고려

## 현황

공지사항 작성 시 이미지를 base64로 변환해 HTML 문자열 안에 통째로 박아 DB에 저장하고 있다.

```html
<!-- 현재: DB에 저장되는 HTML -->
<img src="data:image/jpeg;base64,/9j/4AAQSkZJRgAB...">
```

## 문제점

- DB 용량을 과도하게 사용 (이미지 데이터가 텍스트로 저장됨)
- 이미지를 별도 리소스로 캐시할 수 없음 — API 응답 전체를 캐시하거나 안 하거나 둘 중 하나
- 일반적인 웹사이트는 이미지를 파일 스토리지(S3 등)에 저장하고 DB에는 URL만 기록함

```html
<!-- 일반적인 방식 -->
<img src="https://cdn.example.com/images/abc123.jpg">
```

## 개선 방향

이미지 업로드 엔드포인트를 별도로 만들고, S3 또는 로컬 파일 스토리지에 저장 후 URL만 DB에 기록하는 방식으로 전환.

## 발견 경위
