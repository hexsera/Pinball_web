# 권한 검증 미비 이슈

## 상태
미해결

## 개요
어드민 페이지 및 사용자 관련 API에서 권한 확인이 충분히 이루어지지 않고 있다.

## 세부 내용

- **어드민 페이지**: 클라이언트 측에서 role 확인만으로 접근 제어 — 서버 측 권한 검증 없음
- **User 테이블 API**: 일부 엔드포인트에서 인증 없이 또는 본인 확인 없이 타 사용자 데이터 조회/수정 가능
- **관리자 전용 API**: admin role 검사가 누락되었거나 일관되지 않음

## 해결 방향 (미정)

- API 엔드포인트마다 서버 측 role/identity 검증 미들웨어 또는 의존성 추가
- 어드민 전용 엔드포인트에 `require_admin` 의존성 적용
- 일반 사용자가 자신의 리소스만 접근 가능하도록 `user_id` 일치 검증 추가

## 관련 영역

- `backend/routers/` — 각 라우터의 엔드포인트
- `backend/dependencies.py` (또는 동등한 인증 의존성 파일)
- 프론트엔드 어드민 페이지 라우트 가드
