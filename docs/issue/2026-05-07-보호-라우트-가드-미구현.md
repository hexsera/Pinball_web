# 보호 라우트 가드 미구현

- 발견일: 2026-05-07
- 관련 작업: JWT 5단계 (localStorage 의존 제거)

## 문제

Refresh Token이 없거나 만료된 상태에서 보호가 필요한 페이지(대시보드, 마이페이지 등)에 직접 접근해도 `/login`으로 리다이렉트되지 않는다.

`AuthContext`의 초기화 시 refresh 실패 처리가 비로그인 상태 유지만 하고 리다이렉트를 하지 않기 때문이다. 초기화 시점에 무조건 `/login`으로 보내면 로그인 불필요 페이지(홈, 공지사항 등)도 영향을 받으므로, 보호 라우트에서만 선택적으로 처리해야 한다.

## 해결 방향

`App.jsx`에 `PrivateRoute` 컴포넌트를 추가해 보호가 필요한 라우트를 래핑한다. `loading` 중엔 대기, `isLoggedIn`이 false면 `/login`으로 리다이렉트.

```jsx
function PrivateRoute({ children }) {
  const { isLoggedIn, loading } = useAuth();
  if (loading) return null;
  if (!isLoggedIn) return <Navigate to="/login" replace />;
  return children;
}
```

보호가 필요한 라우트: `/dashboard`, `/pinball`, `/admin/*`, `/user/*`
