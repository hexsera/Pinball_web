# HomePage 월간 랭킹 로딩 상태 없음

- 발견일: 2026-05-18
- 해결일: 2026-05-19
- 관련 파일: `frontend/src/pages/HomePage/HomePage.jsx`, `frontend/src/main.jsx`

## 증상

홈페이지에 월간 랭킹 테이블이 있어 자주 조회되고 api 를 불러올 때 로딩 UI 가 없다.


## 원인 분석

`useEffect` + `useState` + `axios`로 직접 fetch하는 방식을 사용했다. 로딩 상태를 별도 `useState`로 관리하지 않아 데이터 수신 전 빈 테이블이 노출됐다. 또한 페이지를 나갔다 다시 들어오면 캐시 없이 매번 재요청이 발생했다.

## 무엇을 하려고 했는지
우선 해당 api 를 캐싱하려고 한다. 캐싱방법에는 3가지 (reactquery, Cache-Control, Etag) 가 있다. 각각 reactquery 는 메모리에 캐시, Cache-Control 은 브라우저에 캐시, Etag 는 서버와 클라이언트가 협업하는 캐시형태이다.
Cache-Control 은 js 에서 캐시 제어를 하기 힘들고 reactquery 는 캐시 컨트롤을 할 수 있기 때문에 reactquery 를 선택했다.

reactquery 는 api 응답을 관리해주는 패키지 이다. 크게 api 응답에 대한 상태와 캐시 관리를 해준다고 생각하면 된다.

reactquery 로 api 를 캐싱하고, 응답이 오기전에 로딩 UI 를 제어하고, 오래 머무르고 있으면 자동으로 랭킹이 업데이트 되도록 하게 했다.



## 해결 방법

`@tanstack/react-query`를 도입해 `useQuery`로 교체했다.

### 변경 전

```jsx
// HomePage.jsx
const [ranking, setRanking] = useState([]);
const [rankingError, setRankingError] = useState(false);

useEffect(() => {
  axios.get('/api/v1/monthly-scores')
    .then(res => setRanking(res.data.scores.slice(0, 10)))
    .catch(() => setRankingError(true));
}, []);
```

### 변경 후

```jsx
// main.jsx — QueryClientProvider로 앱 감싸기
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
const queryClient = new QueryClient();

ReactDOM.createRoot(document.getElementById('root')).render(
  <QueryClientProvider client={queryClient}>
    <App />
  </QueryClientProvider>
);

// HomePage.jsx — useQuery 적용
const { data, isPending, isError } = useQuery({
  queryKey: ['monthly-scores'],
  queryFn: () => axios.get('/api/v1/monthly-scores').then(res => res.data.scores.slice(0, 10)),
  staleTime: 1000 * 60 * 2,    // 2분간 캐시 신선 유지
  refetchInterval: 1000 * 30,  // 30초마다 자동 재요청
});
```

## 적용 효과

처음 접속할때 api 를 요청해준 후 캐시 시간 안에 다른곳으로 갔다 와도 api 요청을 안보낸다.
만약 홈페이지에서 계속 머무르고 있으면 30초 마다 재 요청을 보낸다.
크롬 개발자도구 network 에서 3G 상태(느린 네트워크 속도) 일때 api 응답 도달 전 "불러오는중..." 텍스트가 뜬다.


## 수정된 파일

- `frontend/package.json` — `@tanstack/react-query` 의존성 추가
- `frontend/src/main.jsx` — `QueryClientProvider` 추가
- `frontend/src/pages/HomePage/HomePage.jsx` — `useQuery` 적용, `useEffect`/`useState` 제거
