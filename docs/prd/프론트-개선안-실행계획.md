# 프론트 개선안 실행계획

## 요구사항 요약

**요구사항**: HomeHeader 바로가기 섹션 정렬/동작 개선 및 NoticeDetailPage UI 개선

**목적**: 헤더의 레이아웃 버그와 토글 상태 버그를 수정하고, NoticeDetailPage를 일관된 디자인 시스템에 맞게 개선한다.

## 현재상태 분석

- `HomeHeader.jsx`: 바로가기 섹션이 좌측 로고·버튼 그룹 안에 포함되어 있어 가운데 정렬이 안 됨. 로고 클릭 이벤트 없음. 다른 페이지로 이동 시 `activeItem` 상태가 초기화되지 않아 토글 효과가 유지됨. 아이템 활성 시 글자 색이 `COLORS.primary`(보라)로 변경되어 흰색이 아님.
- `NoticeDetailPage.jsx`: 이미 `HomeHeader`를 사용 중. 제목이 Paper 밖 독립 `Typography`로 존재하지 않고 Paper 안에 있음. 글 작성 영역(content Box)에 고정 높이나 overflow 처리 없음.

## 구현 방법

- `HomeHeader`: Toolbar를 3-column flex 레이아웃(`flex: 1`)으로 변경하여 바로가기 섹션을 중앙 배치. 로고에 `onClick` 추가. `useLocation`으로 현재 경로를 감지해 `activeItem`을 경로 기반으로 계산(state 제거). 활성 항목 글자색을 `COLORS.text`(흰색)로 유지하되 배경/밑줄로만 활성 표시.
- `NoticeDetailPage`: Paper가 제목·날짜·구분선·본문을 모두 포함하도록 구조 유지(이미 포함됨). Paper와 content Box에 색상·높이·overflow 스타일 적용.

## 구현 단계

### 1. HomeHeader — 바로가기 섹션 가운데 정렬

```jsx
<Toolbar sx={{ display: 'flex', justifyContent: 'space-between', alignItems: 'center' }}>
  {/* 왼쪽 */}
  <Box sx={{ display: 'flex', alignItems: 'center', gap: 1, flex: 1 }}>
    <Typography ...>HEXSERA PINBALL</Typography>
    <Button ...>게임하기</Button>
  </Box>

  {/* 중앙 — 바로가기 */}
  <Box sx={{ display: 'flex', alignItems: 'center', gap: 0.5 }}>
    {SHORTCUTS.map(...)}
  </Box>

  {/* 오른쪽 */}
  <Box sx={{ flex: 1, display: 'flex', justifyContent: 'flex-end' }}>
    <HeaderUserInfo ... />
  </Box>
</Toolbar>
```
- Toolbar를 3개 Box로 분리하고 좌우에 `flex: 1`을 주면 중앙 Box가 자동으로 가운데 정렬된다.

### 2. HomeHeader — 로고 클릭 시 홈 이동

```jsx
<Typography
  variant="h6"
  onClick={() => navigate('/')}
  sx={{ cursor: 'pointer', color: COLORS.text, fontWeight: 'bold', ... }}
>
  HEXSERA PINBALL
</Typography>
```
- `onClick`으로 `navigate('/')` 호출, `cursor: 'pointer'`로 클릭 가능 표시.

### 3. HomeHeader — 경로 기반 활성 상태 (토글 버그 수정)

```jsx
import { useNavigate, useLocation } from 'react-router-dom';

function HomeHeader() {
  const navigate = useNavigate();
  const location = useLocation();
  // activeItem state 제거

  const isActive = (path) => location.pathname.startsWith(path);

  const handleShortcut = (item) => {
    navigate(item.path);
  };
  // ...
  sx={{
    color: isActive(item.path) ? COLORS.text : COLORS.text,  // 항상 흰색
    backgroundColor: isActive(item.path) ? 'rgba(79,70,229,0.15)' : 'transparent',
    borderBottom: isActive(item.path) ? `2px solid ${COLORS.primary}` : '2px solid transparent',
  }}
```
- `useState` 대신 `useLocation`으로 현재 경로를 실시간 감지하므로 페이지 이동 후에도 정확히 반영된다.
- 활성/비활성 모두 `color: COLORS.text`(흰색)로 설정하고, 활성 표시는 배경색과 밑줄로만 한다.

### 4. NoticeDetailPage — Paper 색상 및 글 작성 영역 높이/스크롤

```jsx
<Box sx={{ minHeight: '100vh', backgroundColor: COLORS.bg, display: 'flex', flexDirection: 'column' }}>
  <HomeHeader />
  <Container maxWidth="md" sx={{ py: 4, flex: 1, display: 'flex', flexDirection: 'column' }}>
    <Button ...>← 목록으로</Button>
    <Paper sx={{
      backgroundColor: '#1E293B',
      border: `1px solid ${COLORS.border}`,
      p: 3,
      flex: 1,
      display: 'flex',
      flexDirection: 'column',
      overflow: 'hidden',
    }}>
      <Typography variant="h5" sx={{ color: COLORS.text, mb: 1 }}>{notice.title}</Typography>
      <Typography sx={{ color: COLORS.subText, fontSize: '0.85rem', mb: 2 }}>...</Typography>
      <Divider sx={{ borderColor: COLORS.border, mb: 2 }} />
      <Box sx={{ color: COLORS.text, flex: 1, overflowY: 'auto', '& img': { maxWidth: '100%' } }}
           dangerouslySetInnerHTML={{ __html: notice.content }} />
      {/* admin 버튼 */}
    </Paper>
  </Container>
</Box>
```
- 최상위 Box와 Container, Paper에 `flex` 레이아웃을 연결해 Paper가 남은 뷰포트 높이를 채우게 한다.
- 본문 Box에 `flex: 1`과 `overflowY: 'auto'`를 적용해 내용 초과 시 내부 스크롤이 생긴다.
