## 목적
친구 추가 조회, 승인 api 엔드포인트를 생성한다.

## 이유
사용자간의 커뮤니케이션을 위해 친구 네트워크가 필요하기 때문에.

## 구조
/api/friend-requests 는 친구 요청 api 이다. 이에 맞는 조회, 승인 api 엔드포인트를 생성해야한다.

친구요청을 임시 메모리(배열) 에 저장시킨다.

id 는 친구 요청측 id, requester_id 는 친구 요청받는 id 이다.

테이블을 생성하지 않는다.
보안을 고려하지 않는다.
검증 로직을 고려하지 않는다.

친구 추가 목록을 조회, 친구 추가를 승인 하거나 거절하도록한다.
