# 관리자 통계 페이지 실행계획

## 요구사항 요약

**요구사항**: Admin 페이지에 통계 페이지를 추가하고, 사이드바 메뉴에서 접근 가능하도록 구현

**목적**: 관리자가 시스템의 통계 데이터를 확인할 수 있는 전용 페이지 제공

## 현재상태 분석

- Admin 페이지 구조: AdminPage, AdminUserPage가 존재
- 공통 컴포넌트: AdminHeader, AdminSidebar 사용
- AdminSidebar의 menuItems: '대시보드', '회원관리' 2개 메뉴 존재
- 라우팅: App.jsx에 `/admin`, `/admin/users` 경로 설정됨
- AdminSidebar는 React Router의 useNavigate, useLocation으로 페이지 이동 및 메뉴 선택 상태 관리

## 구현 방법

- AdminUserPage.jsx를 참고하여 AdminStatisticsPage.jsx 생성
- AdminPage와 동일한 레이아웃 구조 사용 (Flexbox로 Header, Sidebar, Main 조합)
- AdminStatisticsMain.jsx 컴포넌트 생성 (통계 페이지 메인 콘텐츠)
- AdminSidebar.jsx의 menuItems 배열에 '통계' 메뉴 추가
- App.jsx에 `/admin/statistics` 경로 추가
- Material-UI의 BarChart 아이콘 사용

- **무엇을 하는가**: 통계 페이지의 메인 콘텐츠 영역을 담당하는 컴포넌트 생성
- AdminMain.jsx와 동일한 레이아웃 구조 사용 (mt: 8로 헤더 공간 확보, ml: '270px'로 사이드바 공간 확보)
- 색상 체계: Dashboard와 동일 (#F9FAFB 배경, #ffffff 카드, #e5e7eb 보더)
- component="main"으로 시맨틱 HTML 적용
