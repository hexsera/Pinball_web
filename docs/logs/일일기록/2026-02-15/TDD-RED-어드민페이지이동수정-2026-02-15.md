# TDD RED 계획: 어드민 페이지 이동 수정

## 구현 목표
HeaderUserInfo.jsx 에서 "페이지 이동" 메뉴 항목을 수정한다.
- role이 'admin'인 사용자에게만 표시
- role은 API 호출(`GET /api/v1/users/{id}`)로 가져옴
- 어드민 페이지(`/admin`으로 시작)에 있으면 → "메인 페이지 이동"
- 어드민 페이지가 아니면 → "관리 페이지 이동"

## RED 테스트 목록

1. `role이 admin인 사용자는 메뉴에 "관리 페이지 이동"이 표시된다`
   - admin role을 가진 유저로 로그인 시, 아바타 메뉴를 열면 "관리 페이지 이동" 항목이 보인다

2. `role이 user인 사용자는 메뉴에 페이지 이동 항목이 표시되지 않는다`
   - role이 'user'인 유저로 로그인 시, 아바타 메뉴를 열면 페이지 이동 관련 항목이 없다

3. `role이 없는 사용자는 메뉴에 페이지 이동 항목이 표시되지 않는다`
   - role 필드가 없는 유저로 로그인 시, 아바타 메뉴에 페이지 이동 관련 항목이 없다

4. `admin 유저가 /admin 경로에 있으면 "메인 페이지 이동"이 표시된다`
   - admin 유저가 /admin 경로에서 메뉴를 열면 "메인 페이지 이동" 항목이 보인다

5. `admin 유저가 /admin 경로가 아니면 "관리 페이지 이동"이 표시된다`
   - admin 유저가 / 경로에서 메뉴를 열면 "관리 페이지 이동" 항목이 보인다

6. `컴포넌트 마운트 시 API로 role을 가져온다`
   - 로그인 상태에서 컴포넌트가 마운트될 때 GET /api/v1/users/{id} 호출이 일어난다

7. `"관리 페이지 이동" 클릭 시 /admin으로 이동한다`
   - admin 유저가 / 경로에서 "관리 페이지 이동" 클릭 시 /admin으로 이동한다

8. `"메인 페이지 이동" 클릭 시 /로 이동한다`
   - admin 유저가 /admin 경로에서 "메인 페이지 이동" 클릭 시 /로 이동한다

## 변경이 필요한 기존 테스트
기존 테스트 중 role 없이 "페이지 이동" 텍스트를 검증하는 테스트는
새 요구사항(admin만 보임, 문구 변경)과 충돌하므로 함께 수정이 필요하다:
- `메뉴에 "페이지 이동" MenuItem이 존재한다` → 삭제 또는 수정
- `메인 페이지(/)에서 "페이지 이동" 클릭 시 /admin으로 이동한다` → 수정
- `/admin으로 시작하는 페이지에서 "페이지 이동" 클릭 시 /로 이동한다` → 수정
