# TDD GREEN 완료: 어드민 페이지 이동 수정

## 구현 완료 항목
- `HeaderUserInfo.jsx`에서 컴포넌트 마운트 시 `GET /api/v1/users/{id}` API 호출로 role 가져오기
- role이 'admin'인 경우에만 페이지 이동 MenuItem 표시
- 현재 경로가 `/admin`으로 시작하면 "메인 페이지 이동", 아니면 "관리 페이지 이동" 표시

## 통과한 테스트 목록 (15개)
1. 아바타 메뉴를 열면 사용자 이메일이 표시된다
2. 아바타 메뉴에 닫기 버튼이 있다
3. 닫기 버튼 클릭 시 메뉴가 닫힌다
4. 메뉴에 "계정설정" MenuItem이 존재한다
5. "계정설정" 클릭 시 /dashboard로 이동하고 section이 "계정"이다
6. 메뉴에 "친구" MenuItem이 존재한다
7. "친구" 클릭 시 /dashboard로 이동하고 section이 "친구"이다
8. role이 admin인 사용자는 메뉴에 "관리 페이지 이동"이 표시된다
9. role이 user인 사용자는 메뉴에 페이지 이동 항목이 표시되지 않는다
10. role이 없는 사용자는 메뉴에 페이지 이동 항목이 표시되지 않는다
11. admin 유저가 /admin 경로에 있으면 "메인 페이지 이동"이 표시된다
12. admin 유저가 /admin 경로가 아니면 "관리 페이지 이동"이 표시된다
13. 컴포넌트 마운트 시 GET /api/v1/users/{id}를 호출한다
14. "관리 페이지 이동" 클릭 시 /admin으로 이동한다
15. "메인 페이지 이동" 클릭 시 /로 이동한다
