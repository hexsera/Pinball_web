# 헤더 바로가기 섹션 실행계획

## 요구사항 요약

**요구사항**: HomeHeader에 바로가기 섹션을 추가하고, 공지사항(notice) 아이템을 포함한다. 아이템에 hover 효과와 클릭 시 toggle 효과를 적용한다. NoticeListPage와 NoticeDetailPage의 기존 헤더를 HomeHeader로 교체한다.

**목적**: 헤더에서 공지사항으로 빠르게 이동할 수 있도록 하고, 추후 바로가기 아이템 확장을 대비한다.

## 현재상태 분석

- `HomeHeader.jsx`: AppBar 기반. 로고, 게임하기 버튼, HeaderUserInfo로 구성. 바로가기 섹션 없음.
- `NoticeListPage.jsx`: 자체 AppBar 인라인 구현 (로고 클릭 + HeaderUserInfo만 포함).
- `NoticeDetailPage.jsx`: NoticeListPage와 동일한 자체 AppBar 인라인 구현.
- `HomePage.jsx`: `HomeHeader` 컴포넌트를 import해서 사용 중.

## 구현 방법

- HomeHeader에 바로가기 아이템 배열을 상수로 정의 → 추후 아이템 추가 시 배열에만 추가하면 됨.
- 각 아이템은 `active` 상태(toggle)를 관리하기 위해 `activeItem` state로 관리.
- hover 효과는 MUI `sx`의 `'&:hover'` 속성으로 구현.
- toggle 효과는 클릭한 아이템의 label을 `activeItem` state에 저장하고, 동일 아이템 재클릭 시 해제.
- NoticeListPage, NoticeDetailPage의 인라인 AppBar를 제거하고 `HomeHeader`로 교체.

## 구현 단계

### 1. HomeHeader에 바로가기 아이템 배열 및 상태 정의

```jsx
const SHORTCUTS = [
  { label: '공지사항', path: '/notice' },
];

function HomeHeader() {
  const navigate = useNavigate();
  const [activeItem, setActiveItem] = useState(null);

  const handleShortcut = (item) => {
    setActiveItem(prev => prev === item.label ? null : item.label);
    navigate(item.path);
  };
  // ...
}
```
- **무엇을 하는가**: 바로가기 목록을 배열로 분리해 추후 아이템 추가 시 배열에만 추가하면 되도록 확장성을 확보.
- `activeItem`은 현재 toggle된 아이템의 label 값을 저장. 동일 아이템 클릭 시 null로 초기화.

### 2. HomeHeader에 바로가기 섹션 렌더링 (hover + toggle 스타일)

```jsx
<Box sx={{ display: 'flex', alignItems: 'center', gap: 0.5 }}>
  {SHORTCUTS.map((item) => (
    <Button
      key={item.label}
      onClick={() => handleShortcut(item)}
      sx={{
        color: activeItem === item.label ? COLORS.primary : COLORS.text,
        backgroundColor: activeItem === item.label ? 'rgba(79,70,229,0.15)' : 'transparent',
        borderBottom: activeItem === item.label ? `2px solid ${COLORS.primary}` : '2px solid transparent',
        borderRadius: 0,
        px: 1.5,
        '&:hover': {
          backgroundColor: 'rgba(79,70,229,0.1)',
          color: COLORS.primary,
        },
        transition: 'all 0.15s ease',
      }}
    >
      {item.label}
    </Button>
  ))}
</Box>
```
- **무엇을 하는가**: SHORTCUTS 배열을 map으로 순회해 바로가기 버튼을 동적으로 렌더링.
- `activeItem === item.label` 조건으로 toggle 상태에 따라 배경색, 텍스트 색, 하단 border가 변경됨.
- `'&:hover'` MUI sx 속성으로 마우스 올릴 때 배경/텍스트 색 변경.

### 3. HomeHeader Toolbar 레이아웃에 바로가기 섹션 삽입

```jsx
<Toolbar sx={{ justifyContent: 'space-between' }}>
  <Box sx={{ display: 'flex', alignItems: 'center', gap: 1 }}>
    <Typography ...>HEXSERA PINBALL</Typography>
    <Button ...>게임하기</Button>
    {/* 바로가기 섹션 */}
    <Box sx={{ display: 'flex', alignItems: 'center', gap: 0.5, ml: 1 }}>
      {SHORTCUTS.map(...)}
    </Box>
  </Box>
  <HeaderUserInfo ... />
</Toolbar>
```
- **무엇을 하는가**: 게임하기 버튼 우측에 바로가기 섹션을 배치해 로고/CTA와 함께 왼쪽 영역에 정렬.

### 4. NoticeListPage 인라인 AppBar를 HomeHeader로 교체

```jsx
// 제거할 코드 (NoticeListPage.jsx)
import { AppBar, Toolbar, Typography } from '@mui/material';
// <AppBar>...</AppBar> 블록 전체 제거

// 추가할 코드
import HomeHeader from '../../components/HomeHeader';
// ...
return (
  <Box sx={{ minHeight: '100vh', backgroundColor: COLORS.bg }}>
    <HomeHeader />
    <Container maxWidth="md" sx={{ py: 4 }}>
      {/* 기존 콘텐츠 */}
    </Container>
  </Box>
);
```
- **무엇을 하는가**: 중복된 인라인 AppBar 코드를 제거하고 공통 HomeHeader 컴포넌트로 통일.
- AppBar 관련 MUI import(AppBar, Toolbar, Typography)가 더 이상 필요 없으면 제거.
