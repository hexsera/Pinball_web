# 프론트 개선안 실행계획

## 요구사항 요약

**요구사항**:
1. HomeHeader - 바로가기 섹션 가운데 정렬
2. HomeHeader - "HEXSERA PINBALL" 클릭 시 "/" 이동
3. HomeHeader - 다른 페이지로 이동 후 바로가기 토글 상태 초기화 문제 수정
4. HomeHeader - 바로가기 클릭 활성화 상태일 때 글자 색상 흰색 유지
5. NoticeWritePage - 헤더를 HomeHeader로 교체
6. NoticeWritePage - Paper 및 글 작성 영역 배경색 #1E293B
7. NoticeWritePage - Paper가 제목(TextField) 포함
8. NoticeWritePage - 글 작성 영역이 뷰포트 높이에 맞게 늘어나고, 내용 넘치면 내부 스크롤

**목적**: 헤더 UI 일관성 확보, 사용자 경험 개선, 글 작성 페이지 레이아웃 완성

## 현재상태 분석

**HomeHeader.jsx (현재 문제)**:
- `HEXSERA PINBALL` 텍스트는 클릭 이벤트 없음 (navigate 미연결)
- 바로가기 섹션이 좌측 정렬 (Toolbar가 `justifyContent: 'space-between'`이나 바로가기가 좌측 Box 안에 있음)
- `activeItem` 상태가 컴포넌트 내부 state — 페이지 이동 후에도 초기화되지 않음
- 활성 버튼 글자 색상이 `COLORS.primary` (인디고)로 설정되어 있어 흰색이 아님

**NoticeWritePage.jsx (현재 문제)**:
- 독립적인 AppBar/Toolbar를 직접 렌더링 (HomeHeader 미사용)
- `Paper` 배경색이 `#fff` (흰색)이며 제목(TextField)이 Paper 외부에 존재
- 글 작성 영역(`EditorContent`)에 높이 제한 없어 내용이 늘어나도 스크롤 없음

## 구현 방법

- **HomeHeader 토글 문제**: `useLocation`으로 현재 경로를 감지해 `activeItem`을 경로 기반으로 동기화
- **바로가기 가운데 정렬**: Toolbar 내 좌측 Box를 `flex: 1`로 확장, 바로가기 섹션을 별도 Box로 분리해 `position: absolute`나 `mx: auto`로 가운데 배치
- **NoticeWritePage 리팩터**: HomeHeader import 후 기존 AppBar 제거, Paper로 제목+에디터 감싸기, Paper 높이를 `calc(100vh - Npx)` 기반으로 설정, 에디터 영역에 `overflow-y: auto`

## 구현 단계

### 1. HomeHeader - "HEXSERA PINBALL" 클릭 시 "/" 이동
```jsx
<Typography
  variant="h6"
  onClick={() => navigate('/')}
  sx={{ color: COLORS.text, fontWeight: 'bold', cursor: 'pointer', ... }}
>
  HEXSERA PINBALL
</Typography>
```
- **무엇을 하는가**: 로고 텍스트를 홈으로 이동하는 링크로 만듦
- `onClick`에 `navigate('/')` 추가, `cursor: 'pointer'` 추가

### 2. HomeHeader - 바로가기 섹션 가운데 정렬
```jsx
<Toolbar sx={{ position: 'relative', justifyContent: 'space-between' }}>
  <Box sx={{ display: 'flex', alignItems: 'center', gap: 1 }}>
    {/* 로고 + 게임하기 버튼 */}
  </Box>

  {/* 바로가기 섹션: Toolbar 중앙 고정 */}
  <Box sx={{
    position: 'absolute',
    left: '50%',
    transform: 'translateX(-50%)',
    display: 'flex', alignItems: 'center', gap: 0.5
  }}>
    {SHORTCUTS.map(...)}
  </Box>

  <HeaderUserInfo ... />
</Toolbar>
```
- **무엇을 하는가**: 바로가기 섹션을 Toolbar 정중앙에 고정
- `position: absolute` + `left: 50%` + `transform: translateX(-50%)`로 완전한 가운데 정렬 구현
- 좌우 요소(로고, 유저 정보)가 바로가기 영역에 겹치지 않게 분리

### 3. HomeHeader - 토글 상태 경로 동기화 (이동 후 초기화)
```jsx
import { useLocation } from 'react-router-dom';

function HomeHeader() {
  const location = useLocation();
  const activeItem = SHORTCUTS.find(s => location.pathname.startsWith(s.path))?.label ?? null;

  const handleShortcut = (item) => {
    navigate(item.path);
  };
  // activeItem을 useState 대신 location 기반으로 파생
```
- **무엇을 하는가**: `activeItem`을 로컬 state가 아닌 현재 URL 경로(`location.pathname`)에서 파생
- 페이지 이동 시 URL이 바뀌므로 자동으로 올바른 활성 상태가 반영됨
- `useState` + `setActiveItem` 제거

### 4. HomeHeader - 활성 바로가기 글자 색상 흰색 유지
```jsx
sx={{
  color: COLORS.text,  // 항상 흰색 (기존: activeItem === item.label ? COLORS.primary : COLORS.text)
  backgroundColor: activeItem === item.label ? 'rgba(79,70,229,0.15)' : 'transparent',
  borderBottom: activeItem === item.label ? `2px solid ${COLORS.primary}` : '2px solid transparent',
  '&:hover': {
    backgroundColor: 'rgba(79,70,229,0.1)',
    color: COLORS.text,  // hover도 흰색 유지
  },
}}
```
- **무엇을 하는가**: 활성/hover 상태에서 글자 색상을 `COLORS.text`(흰색)으로 고정
- 활성 표시는 `borderBottom` + `backgroundColor`로만 구분

### 5. NoticeWritePage - HomeHeader로 교체
```jsx
import HomeHeader from '../../components/HomeHeader';

// 기존 AppBar/Toolbar 블록 제거 후:
return (
  <Box sx={{ minHeight: '100vh', backgroundColor: COLORS.bg, display: 'flex', flexDirection: 'column' }}>
    <HomeHeader />
    <Container maxWidth="md" sx={{ py: 4, flex: 1, display: 'flex', flexDirection: 'column' }}>
      ...
    </Container>
  </Box>
);
```
- **무엇을 하는가**: 중복된 AppBar 코드를 제거하고 공통 HomeHeader 컴포넌트로 대체
- `Box`에 `display: flex, flexDirection: column`을 추가해 이후 높이 확장 레이아웃에 대비

### 6. NoticeWritePage - Paper가 제목 포함 + 색상 변경 + 에디터 스크롤
```jsx
<Paper sx={{
  backgroundColor: COLORS.card,  // #1E293B
  p: 3,
  mb: 2,
  flex: 1,
  display: 'flex',
  flexDirection: 'column',
  overflow: 'hidden',
}}>
  <TextField
    fullWidth
    label="제목"
    value={title}
    onChange={e => setTitle(e.target.value)}
    sx={{ mb: 2, input: { color: COLORS.text }, label: { color: COLORS.subText },
          '& .MuiOutlinedInput-root': { '& fieldset': { borderColor: COLORS.border } } }}
  />
  {/* 툴바 버튼들 */}
  <Box sx={{ mb: 1, display: 'flex', gap: 1, flexWrap: 'wrap' }}>...</Box>
  {/* 에디터 영역: 남은 공간 채우고 스크롤 */}
  <Box sx={{ flex: 1, overflowY: 'auto', color: COLORS.text }}>
    <EditorContent editor={editor} />
  </Box>
</Paper>
```
- **무엇을 하는가**: TextField(제목)를 Paper 안으로 이동, Paper 배경색을 `#1E293B`로 변경
- Paper에 `flex: 1`을 부여해 뷰포트 세로 길이에 맞게 늘어나게 설정
- 에디터를 감싼 Box에 `flex: 1 + overflowY: auto`로 내용이 넘칠 때 내부 스크롤 적용
