# TDD RED 계획: 공지사항 프론트엔드 (섹션 2, 4)

## 구현 목표

- 섹션 2: `noticeService.js` — 공지사항 API 호출 서비스 레이어 5개 함수
- 섹션 4: `NoticeWritePage.jsx` — Tiptap 에디터 기반 공지사항 작성 페이지

> 섹션 1 (Tiptap 패키지 설치)은 TDD 대상이 아닌 빌드 환경 설정이므로 테스트 코드 작성 전 패키지 설치를 선행한다.

## RED 테스트 목록

### 파일 1: `frontend/src/test/noticeService.test.js`

#### `getNotices`
1. `getNotices — GET /api/v1/notices를 호출한다`: `axios.get('/api/v1/notices')` 호출 여부 확인
2. `getNotices — 서버 응답 data 배열을 반환한다`: `res.data` 반환 확인

#### `getNotice`
3. `getNotice — GET /api/v1/notices/{id}를 호출한다`: `axios.get('/api/v1/notices/5')` 호출 여부 확인
4. `getNotice — 서버 응답 data 객체를 반환한다`: `res.data` 반환 확인

#### `createNotice`
5. `createNotice — POST /api/v1/notices를 title, content와 함께 호출한다`: body에 `{ title, content }` 포함 확인
6. `createNotice — 서버 응답 data를 반환한다`: `res.data` 반환 확인

#### `deleteNotice`
7. `deleteNotice — DELETE /api/v1/notices/{id}를 호출한다`: `axios.delete('/api/v1/notices/3')` 호출 여부 확인
8. `deleteNotice — 서버 응답 data를 반환한다`: `res.data` 반환 확인

#### `uploadNoticeImage`
9. `uploadNoticeImage — POST /api/v1/notices/upload-image를 FormData로 호출한다`: FormData에 `file` 키 포함 확인
10. `uploadNoticeImage — 서버 응답 data를 반환한다`: `res.data` 반환 확인

### 파일 2: `frontend/src/test/NoticeWritePage.test.jsx`

#### 접근 제어
11. `비로그인 상태에서 /notice로 리다이렉트된다`: `user=null`일 때 `navigate('/notice')` 호출 확인
12. `일반 사용자(role=user) 접근 시 /notice로 리다이렉트된다`: `user.role='user'`일 때 `navigate('/notice')` 호출 확인
13. `admin 사용자는 페이지가 렌더링된다`: `user.role='admin'`일 때 "공지사항 작성" 텍스트 노출 확인

#### 렌더링
14. `제목 입력 필드가 존재한다`: `label="제목"` TextField 렌더링 확인
15. `Bold(B) 툴바 버튼이 존재한다`: "B" 버튼 렌더링 확인
16. `Italic(I) 툴바 버튼이 존재한다`: "I" 버튼 렌더링 확인
17. `H2 툴바 버튼이 존재한다`: "H2" 버튼 렌더링 확인
18. `이미지 업로드 버튼이 존재한다`: "이미지" 버튼 렌더링 확인
19. `저장 버튼이 존재한다`: "저장" 버튼 렌더링 확인
20. `취소 버튼이 존재한다`: "취소" 버튼 렌더링 확인

#### 인터랙션
21. `취소 버튼 클릭 시 /notice로 이동한다`: `navigate('/notice')` 호출 확인
22. `저장 버튼 클릭 시 createNotice가 호출된다`: `createNotice(title, content)` 호출 확인
23. `저장 후 /notice로 이동한다`: createNotice 완료 후 `navigate('/notice')` 호출 확인
