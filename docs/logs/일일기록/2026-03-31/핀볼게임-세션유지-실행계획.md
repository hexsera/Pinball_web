# 핀볼게임 세션 유지 실행계획

## 요구사항 요약

**요구사항**: 핀볼 게임 플레이 중 웹이 끊기거나 탭을 닫고 다시 접속했을 때, 이전 게임 상태(점수, 목숨, 스테이지)를 복원하여 계속 플레이할 수 있도록 한다.

**목적**: 네트워크 불안정이나 실수로 탭을 닫아도 진행 중인 게임을 잃지 않기 위함.

## 현재상태 분석

- 게임 상태(score, lives, stage)는 React state와 useRef로만 관리되며, 브라우저를 닫으면 모두 사라진다.
- 게임 오버 시에만 `POST /api/v1/monthly-scores`로 최종 점수를 PostgreSQL에 저장한다.
- Redis나 별도 NoSQL DB가 없으며, docker-compose에는 PostgreSQL과 MySQL만 존재한다.
- 로그인 정보는 localStorage에 저장되어 있어 재접속 시 사용자 식별은 가능하다.
- `pinballRestart.js`의 `getRestartState()`가 게임 재시작 시 상태를 초기화하는 단일 진입점이다.

## 구현 방법

Redis를 세션 저장소로 사용한다. Redis는 인메모리 DB로 빠른 읽기/쓰기가 가능하고, TTL(만료 시간) 설정으로 오래된 세션을 자동 삭제할 수 있어 게임 세션 저장에 업계 표준으로 사용된다.

- **세션 키**: `pinball_session:{user_id}` 형태로 Redis에 JSON 저장
- **저장 시점**: 스테이지 진행, 목숨 감소, 주기적 점수 변경 시 자동 저장
- **복원 시점**: 게임 페이지 진입 시 저장된 세션 확인 후 사용자에게 선택 제공
- **세션 만료**: 24시간 TTL 적용 (하루 지난 세션은 자동 삭제)

## 구현 단계

### 1. Redis 서비스 추가 (docker-compose.yml)

```yaml
redis:
  image: redis:7-alpine
  container_name: redis-server
  restart: always
  networks:
    - web
  volumes:
    - redis_data:/data
  command: redis-server --appendonly yes

volumes:
  redis_data:
```

- `appendonly yes`로 Redis 재시작 시에도 데이터 유지
- `web` 네트워크에 연결하여 fastapi 컨테이너에서 접근 가능

### 2. 백엔드 의존성 추가 및 Redis 연결 설정

```python
# backend/requirements.txt 에 추가
redis==5.0.1

# backend/app/core/redis_client.py (신규 생성)
import redis
import os

redis_client = redis.Redis(
    host=os.getenv("REDIS_HOST", "redis-server"),
    port=6379,
    db=0,
    decode_responses=True
)
```

- `decode_responses=True`로 bytes 대신 str로 자동 변환
- 환경 변수로 호스트를 설정하여 환경별 유연하게 적용

### 3. 게임 세션 API 엔드포인트 작성

```python
# backend/app/api/v1/game_session.py (신규 생성)
import json
from fastapi import APIRouter, HTTPException
from app.core.redis_client import redis_client

router = APIRouter()
SESSION_TTL = 86400  # 24시간

@router.get("/game-session/{user_id}")
def get_game_session(user_id: int):
    data = redis_client.get(f"pinball_session:{user_id}")
    if not data:
        return {"session": None}
    return {"session": json.loads(data)}

@router.post("/game-session/{user_id}")
def save_game_session(user_id: int, body: dict):
    redis_client.setex(
        f"pinball_session:{user_id}",
        SESSION_TTL,
        json.dumps(body)
    )
    return {"ok": True}

@router.delete("/game-session/{user_id}")
def delete_game_session(user_id: int):
    redis_client.delete(f"pinball_session:{user_id}")
    return {"ok": True}
```

- `setex`는 값 저장과 TTL 설정을 동시에 수행
- DELETE는 게임 오버 또는 신규 게임 시작 시 세션 삭제에 사용

### 5. 프론트엔드 세션 저장 훅 작성

```javascript
// frontend/src/pages/Pinball/useGameSession.js (신규 생성)
import axios from 'axios';

export function useGameSession(userId) {
  const saveSession = (score, lives, stage) => {
    if (!userId) return;
    axios.post(`/api/v1/game-session/${userId}`, { score, lives, stage })
      .catch(() => {});  // 저장 실패는 조용히 무시
  };

  const loadSession = async () => {
    if (!userId) return null;
    const res = await axios.get(`/api/v1/game-session/${userId}`);
    return res.data.session;  // null 또는 {score, lives, stage}
  };

  const clearSession = () => {
    if (!userId) return;
    axios.delete(`/api/v1/game-session/${userId}`).catch(() => {});
  };

  return { saveSession, loadSession, clearSession };
}
```

- 저장 실패 시 게임을 방해하지 않도록 에러를 조용히 처리
- 로그아웃 상태(userId 없음)일 때는 동작하지 않음

### 6. Pinball.jsx에 세션 저장/복원 로직 연결

```javascript
// Pinball.jsx 내 수정 부분

// (1) 훅 초기화
const { saveSession, loadSession, clearSession } = useGameSession(user?.id);

// (2) 게임 시작 시 세션 확인 및 복원
useEffect(() => {
  loadSession().then(session => {
    if (session) {
      const resume = window.confirm('이전 게임을 이어서 하시겠습니까?');
      if (resume) {
        setScore(session.score);
        setLives(session.lives);
        setStage(session.stage);
      } else {
        clearSession();
      }
    }
  });
}, []);

// (3) 스테이지 변경, 목숨 감소 시 세션 저장
useEffect(() => {
  if (gameStarted) saveSession(score, lives, stage);
}, [stage, lives]);

// (4) 게임 오버 시 세션 삭제
// submitScore() 함수 내에서 clearSession() 호출
```

- `window.confirm`으로 복원 여부를 사용자에게 선택하게 함
- 세션 저장은 스테이지/목숨 변경 시에만 수행 (점수 변경마다 호출하면 과도한 API 요청 발생)
