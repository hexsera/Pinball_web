# UI 화면 깜빡임 문제 해결

## 현상

페이지 이동 시 잠깐 흰색으로 깜빡이는 현상 발생. 사이트 전체 톤은 검은 계열이나 어드민 페이지는 흰 계열.

## 원인 분석

### 구조 파악
- `index.html`에 배경색 지정 없음 → 브라우저 기본값(흰색) 노출
- 배경색은 각 페이지 컴포넌트에서 개별 적용 중
- SPA이므로 페이지 이동 시 HTML 재로드는 없음
- 라우트 전환 시 새 컴포넌트가 마운트되기 전 짧은 순간 `body` 기본 흰색이 노출됨

### `index.html`에 고정 배경색을 넣지 못하는 이유
- 검은 배경으로 고정 → 어드민(흰 계열) 페이지에서 검정 → 흰색 반대 깜빡임 발생
- 페이지별 배경색이 달라서 고정값으로는 해결 불가

## 논의된 해결 방안

### 방안 1: `useLayoutEffect` 기반 커스텀 훅
```js
// hooks/useBodyBg.js
export function useBodyBg(color) {
  useLayoutEffect(() => {
    document.body.style.backgroundColor = color;
  }, [color]);
}
```
- 각 페이지에서 `useBodyBg('#000')` 또는 `useBodyBg('#fff')` 호출
- `useLayoutEffect`는 DOM 페인트 전에 동기 실행 → 시각적 깜빡임 차단
- `useEffect`는 렌더링 이후 실행이라 한 프레임 깜빡임이 남을 수 있으므로 `useLayoutEffect` 사용 권장

## 결정 사항

- **채택: `useLayoutEffect` 커스텀 훅 방식**
- 전체 일괄 적용 전에 먼저 홈페이지(`/`)와 로그인 페이지(`/login`) 사이에만 적용해 실제 효과 검증
- 검증 후 전체 확대 여부 결정

## 구현 및 검증 결과 (2026-04-03)

### 구현 내용
- `frontend/src/hooks/useBodyBg.js` 생성
- `HomePage`, `Login` 컴포넌트에 `useBodyBg(COLORS.bg)` 호출 추가

### Playwright를 통한 검증
- Playwright MCP를 Claude Code에 연결해 headless 브라우저로 직접 검증
- 홈(`/`) 및 로그인(`/login`) 양쪽 모두 `document.body.style.backgroundColor = rgb(15, 23, 42)` 정상 적용 확인
- 훅 자체는 정상 동작 중

### 실제 브라우저에서 여전히 깜빡임 발생
- 로그인 버튼 클릭 시 및 뒤로가기 시 모두 흰색 깜빡임 잔존
- `index.html`에 `background-color: #0F172A` 직접 지정 테스트 → 초기 로딩 배경색은 정상 적용됐으나 깜빡임은 해소되지 않음
- `index.html` 고정 배경색은 어드민(흰 계열) 페이지 문제로 채택 불가 → 원상복구

### 미해결 원인 추정
- `index.html`에 `#0F172A` 배경색을 지정한 상태에서도 홈 ↔ 로그인 전환 시 흰색 깜빡임이 발생함 → body 배경 공백이 원인이 아님
- body 배경색 위에 **흰색 레이어가 순간적으로 올라오는 것**이 원인으로 추정 (MUI 컴포넌트 기본값, Aurora 마운트 전 공백 등)
- 뒤로가기의 경우 브라우저 BFCache 복원 과정도 원인일 수 있음

## Aurora 컴포넌트 원인 분석 (2026-04-03 추가)

### 가설: Aurora 마운트 전 공백이 원인

Aurora 컴포넌트의 WebGL 렌더러는 `useEffect` 안에서 초기화됨 ([Aurora.jsx:119](../../frontend/src/components/Aurora/Aurora.jsx)).

```
1. 컴포넌트 마운트 → div만 빈 상태로 렌더링 (배경 없음)
2. useEffect 실행 (브라우저 페인트 이후) → WebGL 캔버스 생성 및 첫 프레임 렌더링
```

즉, **컴포넌트 마운트 후 첫 페인트까지 Aurora가 없는 순간**이 존재함.

- `Aurora.css`의 `.aurora-container`에는 배경색이 없음 → WebGL 붙기 전 투명 상태
- 두 페이지 모두 감싸는 Box에 `backgroundColor: COLORS.bg`는 있으나, 깜빡임 색이 흰색인 점으로 보아 MUI 내부 레이어(Paper 등)가 순간 노출될 가능성 있음

### 검증 계획
- `.aurora-container`에 `background-color: #0F172A` 추가
- 이후 홈 ↔ 로그인 전환 시 깜빡임 소멸 여부 확인
- 소멸 → Aurora 마운트 전 공백이 원인으로 확정

## 추가 시도 기록 (2026-04-03)

### 시도 A: Aurora.css 배경색 + bgColor prop 추가

**방향**: Aurora 컴포넌트가 WebGL 초기화 전 투명하게 노출되는 것이 원인이라는 가설. CSS에 `background-color: #0F172A` 추가, 추가로 `bgColor` prop을 만들어 페이지에서 직접 전달하는 방식도 구현.

**결과**: 깜빡임 미해결. Aurora 레이어 자체는 원인이 아님.

---

### 시도 B: MUI ThemeProvider + CssBaseline 배경색 지정

**방향**: MUI `CssBaseline`이 기본 테마 `palette.background.default`(흰색)를 `body`에 주입하는 것이 원인이라는 가설. `createTheme`에 `background.default: '#0F172A'` 지정 후 `ThemeProvider`로 래핑.

**결과**: 빌드 및 브라우저 검증 전 작업 중단. 효과 미검증.

### 미검증 가설 / 다음 시도 후보

- 모든 시도에서 공통적으로 남는 의문: `index.html`에 배경색을 직접 지정해도 깜빡임이 발생한 이력이 있음 → body 배경이 아니라 **그 위에 흰색 레이어가 올라오는 것**이 원인일 가능성
- MUI Paper, Container 등 기본 흰색 컴포넌트가 페이지 전환 순간 잠깐 렌더링되는 것이 원인일 수 있음
- React Router 전환 시 이전/다음 컴포넌트의 렌더링 중첩 구간 문제일 수도 있음
