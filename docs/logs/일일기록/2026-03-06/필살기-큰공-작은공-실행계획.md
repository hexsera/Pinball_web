# 필살기 (큰공/작은공) 실행계획

## 요구사항 요약

**요구사항**: `b` 키 입력 시 큰공(파란색), `s` 키 입력 시 작은공(빨간색)으로 교체하고, 5초 후 원래 공으로 복귀한다.

**목적**: 플레이어가 전략적으로 공의 크기와 질량을 변경하여 게임에 변화를 줄 수 있는 필살기 메커니즘 제공.

## 현재상태 분석

- 공은 `AIPinball.jsx` 299번째 줄에서 `Bodies.circle(60, SHELF_Y-220, 15, {...})`로 생성, `ballRef.current`에 저장됨
- 공의 기본 반지름: `15`, 기본 색상: `#e94560`
- 키 입력은 `handleKeyDown` (880번째 줄) 함수에서 처리 (`ArrowLeft`, `ArrowRight`, `Space`, `b/s/l`, `n` 키 사용 중)
- Matter.js `World.add` / `World.remove`로 월드에 body 추가/제거 가능
- `ballRef`는 충돌 감지(497~558번째 줄)에서 사용되므로 교체 시 `ballRef.current`도 동기화 필수
- `b`, `s` 키는 현재 `setSkillState` UI 상태 전환에 사용 중 — 필살기 실제 발동 로직을 해당 키에 함께 추가

## 구현 방법

- 큰공/작은공 body를 게임 초기화 시 사전 생성해두고 월드에는 추가하지 않음
- `b` / `s` 키 입력 시: 현재 공을 월드에서 제거 → 필살기 공을 현재 위치/속도로 설정 → 월드에 추가 → `ballRef.current` 교체
- `setTimeout` 5000ms 후 역순으로 원래 공으로 복귀
- 필살기 활성 여부를 `useRef`로 관리하여 중복 발동 방지

## 구현 단계

### 1. 필살기 상태 ref 및 사전 생성 body 선언

```javascript
const bigBallRef = useRef(null);
const smallBallRef = useRef(null);
const specialActiveRef = useRef(false);
const specialTimerRef = useRef(null);
```
- **무엇을 하는가**: 필살기 공 body와 활성 상태를 컴포넌트 생명주기 동안 유지하는 ref 선언
- `specialActiveRef`: 필살기 중복 발동 방지용 플래그
- `specialTimerRef`: 5초 타이머 참조 (cleanup 시 `clearTimeout`에 사용)

### 2. 게임 초기화 시 필살기 body 사전 생성 (ballRef 생성 직후)

```javascript
// 큰공 (파란색, 반지름 30, 질량 +0.5)
const bigBall = Bodies.circle(662, SHELF_Y - 20, 30, {
  restitution: 0.8,
  friction: 0,
  frictionAir: 0,
  density: ball.density + 0.5 / (Math.PI * 30 * 30), // 질량 +0.5 보정
  render: { fillStyle: '#3498db' }
});

// 작은공 (빨간색, 반지름 4.5, 질량 -0.5)
const smallBall = Bodies.circle(662, SHELF_Y - 20, 4.5, {
  restitution: 0.8,
  friction: 0,
  frictionAir: 0,
  density: Math.max(ball.density - 0.5 / (Math.PI * 4.5 * 4.5), 0.0001),
  render: { fillStyle: '#e74c3c' }
});

bigBallRef.current = bigBall;
smallBallRef.current = smallBall;
```
- **무엇을 하는가**: 필살기 공 body를 미리 생성해 ref에 저장 (월드에는 추가 안 함)
- Matter.js에서 질량은 `density × area`로 계산되므로, 목표 질량차를 반지름 기반 면적으로 나눠 density 보정
- `density`를 직접 조정하는 이유: `Body.setMass()`는 Matter.js에서 density를 재계산하므로 동등함

### 3. 필살기 발동/복귀 함수 작성

```javascript
const activateSpecial = (specialBall) => {
  if (specialActiveRef.current) return; // 중복 방지
  specialActiveRef.current = true;

  const currentBall = ballRef.current;
  const pos = { ...currentBall.position };
  const vel = { ...currentBall.velocity };
  const angVel = currentBall.angularVelocity;

  World.remove(engine.world, currentBall);
  Body.setPosition(specialBall, pos);
  Body.setVelocity(specialBall, vel);
  Body.setAngularVelocity(specialBall, angVel);
  World.add(engine.world, specialBall);
  ballRef.current = specialBall;

  specialTimerRef.current = setTimeout(() => {
    const pos2 = { ...specialBall.position };
    const vel2 = { ...specialBall.velocity };
    const angVel2 = specialBall.angularVelocity;

    World.remove(engine.world, specialBall);
    Body.setPosition(currentBall, pos2);
    Body.setVelocity(currentBall, vel2);
    Body.setAngularVelocity(currentBall, angVel2);
    World.add(engine.world, currentBall);
    ballRef.current = currentBall;
    specialActiveRef.current = false;
  }, 5000);
};
```
- **무엇을 하는가**: 현재 공을 필살기 공으로 교체하고, 5초 후 원래 공으로 복귀시키는 핵심 함수
- 위치(`position`), 속도(`velocity`), 각속도(`angularVelocity`)를 복사해 물리 연속성 유지
- `ballRef.current`를 교체해야 충돌 감지 코드(497번째 줄~)가 올바른 공을 참조함

### 4. handleKeyDown의 기존 `b` / `s` 키 블록에 필살기 발동 추가

```javascript
// 기존 코드 (909번째 줄)
if (event.key === 'b' || event.key === 'B') {
  setSkillState('big');
  activateSpecial(bigBallRef.current); // 추가
}
if (event.key === 's' || event.key === 'S') {
  setSkillState('small');
  activateSpecial(smallBallRef.current); // 추가
}
```
- **무엇을 하는가**: 기존 UI 상태 전환(`setSkillState`)에 실제 필살기 발동(`activateSpecial`) 로직을 함께 호출
- 기존 `b`/`s` 키 블록에 한 줄씩 추가하는 방식으로 기존 코드 구조 유지

### 5. cleanup에서 타이머 해제

```javascript
// useEffect cleanup (1035번째 줄 근처 return 함수 내)
if (specialTimerRef.current) {
  clearTimeout(specialTimerRef.current);
}
```
- **무엇을 하는가**: 컴포넌트 언마운트 시 진행 중인 필살기 타이머를 정리하여 메모리 누수 방지
