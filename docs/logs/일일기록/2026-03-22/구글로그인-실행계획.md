# 구글 로그인 실행계획

## 처리 흐름도

```
[사용자]
   │
   │ 1. "구글로 로그인" 버튼 클릭
   ▼
[Frontend: @react-oauth/google]
   │
   │ 2. 구글 계정 선택 팝업 열림
   ▼
[Google 서버]
   │
   │ 3. Authorization Code 발급
   ▼
[Frontend]
   │
   │ 4. POST /api/v1/auth/google  { code: "..." }
   ▼
[Backend: FastAPI]
   │
   │ 5. POST https://oauth2.googleapis.com/token
   │    { code, client_id, client_secret, redirect_uri, grant_type }
   ▼
[Google 서버]
   │
   │ 6. Access Token 반환
   ▼
[Backend: FastAPI]
   │
   │ 7. GET https://www.googleapis.com/oauth2/v3/userinfo
   │    Authorization: Bearer {access_token}
   ▼
[Google 서버]
   │
   │ 8. { sub, email, name } 반환
   ▼
[Backend: FastAPI]
   │
   │ 9. DB에서 email 조회 → 없으면 자동 생성
   ▼
[PostgreSQL]
   │
   │ 10. { user_id, email, nickname, role } 반환
   ▼
[Backend: FastAPI]
   │
   │ 11. LoginResponse 반환
   ▼
[Frontend]
   │
   │ 12. localStorage에 사용자 정보 저장 → 홈(/) 이동
   ▼
[사용자]
```

> Access Token은 5~8번 구간(서버-서버)에서만 존재. 브라우저에 노출되지 않음.

---

## 요구사항 요약

**요구사항**: Google OAuth 2.0 Authorization Code Flow를 사용하여 구글 계정으로 로그인 기능 추가. DB에 유저 자동 추가는 아직 구현하지 않는다.

**목적**: 이메일/비밀번호 입력 없이 구글 계정으로 간편하게 로그인하여 사용자 진입 장벽을 낮춘다.

## 현재상태 분석

- **인증 방식**: `POST /api/v1/login`에 email + password 전송 → 백엔드 bcrypt 검증 → localStorage에 `{id, name, role, email}` 저장
- **세션 관리**: JWT 없음, localStorage 기반 평문 저장
- **User 모델**: `password`와 `birth_date` 컬럼이 `nullable=False`로 설정되어 있음
- **현재 OAuth 패키지**: 없음 (frontend: React 19, backend: FastAPI 0.109)
- **환경변수**: 프론트엔드 `VITE_GOOGLE_CLIENT_ID`는 이미 설정됨, 백엔드에는 Google OAuth 관련 변수 없음

## 구현 방법

- **Frontend**: `@react-oauth/google` — React용 Google OAuth 표준 패키지
- **Backend**: `Authlib 1.6.9` — FastAPI OAuth 2.0 표준 라이브러리 + `httpx` (비동기 HTTP 클라이언트, 이미 설치됨)
- **Authlib 활용 범위**: 팝업 방식이므로 Authlib의 리다이렉트 흐름은 사용하지 않고, `AsyncOAuth2Client`로 Token 교환 및 userinfo 조회에 활용
- **보안**: Access Token이 브라우저에 노출되지 않음 (구글 권장 방식인 서버-서버 토큰 교환)

## 필요한 환경변수

실행 전 아래 값을 준비해야 한다:

| 변수명 | 위치 | 설명 |
|--------|------|------|
| `VITE_GOOGLE_CLIENT_ID` | `frontend/.env` | Google Cloud Console에서 발급받은 OAuth 2.0 클라이언트 ID (이미 설정됨) |
| `GOOGLE_CLIENT_ID` | `backend/.env` | 프론트엔드와 동일한 클라이언트 ID |
| `GOOGLE_CLIENT_SECRET` | `backend/.env` | Google Cloud Console에서 발급받은 OAuth 2.0 클라이언트 시크릿 |
| `GOOGLE_REDIRECT_URI` | `backend/.env` | 팝업 방식 고정값: `postmessage` |

## 구현 단계

### Phase 1: Frontend

#### 1. 패키지 설치
```bash
cd frontend
npm install @react-oauth/google
```
- **무엇을 하는가**: React 앱에서 Google OAuth 2.0을 처리하는 공식 패키지를 설치한다
- `@react-oauth/google`은 Google Identity Services 기반으로 2022년 이후 표준 패키지다

#### 2. GoogleOAuthProvider 설정 (`frontend/src/main.jsx`)
```jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { GoogleOAuthProvider } from '@react-oauth/google';
import App from './App.jsx';

const rootElement = document.getElementById('root');
const root = ReactDOM.createRoot(rootElement);

// GoogleOAuthProvider: 앱 전체에서 Google OAuth를 사용할 수 있도록 최상위에 래핑
root.render(
  <GoogleOAuthProvider clientId={import.meta.env.VITE_GOOGLE_CLIENT_ID}>
    <App />
  </GoogleOAuthProvider>
);
```
- **무엇을 하는가**: 앱 전체에서 Google OAuth를 사용할 수 있도록 최상위 컴포넌트를 Provider로 감싼다
- `VITE_GOOGLE_CLIENT_ID`: Vite는 `VITE_` 접두사가 있는 환경변수만 클라이언트에 노출한다

#### 3. Google 로그인 버튼 추가 (`frontend/src/pages/Login/Login.jsx`)
```jsx
import { useGoogleLogin } from '@react-oauth/google';

function Login() {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');
  const [loginError, setLoginError] = useState(false);
  const navigate = useNavigate();
  const { login } = useAuth();

  // 구글 로그인: Authorization Code Flow 설정
  const googleLogin = useGoogleLogin({
    flow: 'auth-code',  // Authorization Code Flow (Access Token이 브라우저에 오지 않음)
    onSuccess: async (codeResponse) => {
      try {
        // codeResponse.code = Authorization Code (Access Token 아님)
        const res = await axios.post('/api/v1/auth/google', {
          code: codeResponse.code,
        });
        if (res.status === 200) {
          // 기존 login() 함수 재사용: localStorage 저장 및 리다이렉트
          login({
            id: res.data.user_id,
            name: res.data.nickname,
            role: res.data.role,
            email: res.data.email,
          });
          navigate(res.data.role === 'admin' ? '/admin' : '/');
        }
      } catch (error) {
        setLoginError(true);
        console.log('구글 로그인 실패:', error.response?.data?.detail);
      }
    },
    onError: () => {
      setLoginError(true);
    },
  });

  // ... 기존 handleLogin 함수 유지 ...

  return (
    // ... 기존 JSX ...
    // 회원가입 버튼 위에 구글 로그인 버튼 추가:

    {/* 구글 로그인 버튼 */}
    <Box sx={{ display: 'flex', justifyContent: 'center', mt: 2 }}>
      <Button
        variant="outlined"
        sx={{
          width: '300px',
          borderColor: '#4285F4',
          color: COLORS.text,
          textTransform: 'none',
          fontSize: '0.95rem',
        }}
        onClick={() => googleLogin()}
      >
        <img
          src="https://developers.google.com/identity/images/g-logo.png"
          alt="Google"
          style={{ width: 20, height: 20, marginRight: 10 }}
        />
        구글로 로그인
      </Button>
    </Box>

    {/* 기존 회원가입 버튼 */}
  );
}
```
- **무엇을 하는가**: 구글 로그인 팝업을 띄우고, Authorization Code만 받아 백엔드로 전송한다
- `flow: 'auth-code'` 설정이 핵심 — 이것이 없으면 Implicit Flow로 동작하여 보안이 약해진다
- 성공 시 기존 `login()` 함수를 재사용하여 localStorage 저장 및 페이지 이동을 처리한다
- 구글 로그인 버튼은 로그인 버튼과 회원가입 버튼 사이에 배치한다

### Phase 2: Backend

#### 4. 패키지 추가 (`backend/requirements.txt`)
```txt
Authlib==1.6.9
```
- **무엇을 하는가**: Authlib는 OAuth 2.0 Token 교환 및 userinfo 조회를 처리하는 라이브러리다
- `httpx`는 이미 `requirements.txt`에 포함되어 있으므로 추가하지 않는다

#### 5. 환경변수 추가 (`backend/app/core/config.py`)
```python
class Settings:
    """애플리케이션 설정"""
    # Database
    DATABASE_URL: str = os.getenv("DATABASE_URL")

    GEMINI_API_KEY: str = os.getenv("GEMINI_API_KEY")

    # Google OAuth
    GOOGLE_CLIENT_ID: str = os.getenv("GOOGLE_CLIENT_ID", "")
    GOOGLE_CLIENT_SECRET: str = os.getenv("GOOGLE_CLIENT_SECRET", "")
    GOOGLE_REDIRECT_URI: str = os.getenv("GOOGLE_REDIRECT_URI", "postmessage")
```
- **무엇을 하는가**: Authorization Code → Token 교환 시 구글이 CLIENT_SECRET으로 백엔드를 검증한다
- `GOOGLE_REDIRECT_URI=postmessage`: 팝업 방식 로그인 시 구글이 요구하는 고정값이다

#### 6. Pydantic 스키마 추가 (`backend/app/schemas/user.py`)
```python
class GoogleLoginRequest(BaseModel):
    """구글 로그인 요청: 프론트엔드에서 전송하는 Authorization Code를 받는다"""
    code: str
```
- **무엇을 하는가**: 프론트엔드에서 전송하는 Authorization Code를 받는 요청 스키마다
- `code`는 구글이 팝업을 통해 발급한 일회용 인가 코드다

#### 7. Google OAuth 엔드포인트 생성 (`backend/app/api/v1/auth.py`)
```python
from authlib.integrations.httpx_client import AsyncOAuth2Client
from app.core.config import settings
from app.schemas.user import GoogleLoginRequest

@router.post("/auth/google", response_model=LoginResponse)
async def google_login(data: GoogleLoginRequest, db: Session = Depends(get_db)):
    """구글 로그인 엔드포인트: Authorization Code를 받아 구글 사용자 정보를 조회한다"""

    # 1단계: Authlib AsyncOAuth2Client로 Authorization Code → Access Token 교환
    async with AsyncOAuth2Client(
        client_id=settings.GOOGLE_CLIENT_ID,
        client_secret=settings.GOOGLE_CLIENT_SECRET,
    ) as client:
        token = await client.fetch_token(
            "https://oauth2.googleapis.com/token",
            code=data.code,
            redirect_uri=settings.GOOGLE_REDIRECT_URI,
            grant_type="authorization_code",
        )

    # 2단계: 발급된 Access Token으로 구글 userinfo 조회
    async with AsyncOAuth2Client(token=token) as client:
        resp = await client.get("https://www.googleapis.com/oauth2/v3/userinfo")

    # 구글 응답 검증
    if resp.status_code != 200:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Failed to get Google user info"
        )

    google_user = resp.json()  # {"sub": "...", "email": "...", "name": "..."}

    # 3단계: DB에서 이메일로 사용자 조회
    user = db.query(User).filter(User.email == google_user["email"]).first()

    # 사용자가 DB에 없으면 401 에러 (자동 생성은 아직 구현하지 않음)
    if not user:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="No account found for this Google email. Please register first."
        )

    # 기존 LoginResponse 형식으로 반환
    return LoginResponse(
        message="Login successful",
        user_id=user.id,
        email=user.email,
        nickname=user.nickname,
        role=user.role
    )
```
- **무엇을 하는가**: Authlib `AsyncOAuth2Client`가 구글과의 Token 교환 요청을 구성하고 처리한다
- `fetch_token()`이 `client_id`, `client_secret`, `code`를 조합해 구글 token 엔드포인트에 POST 전송한다
- token 객체를 그대로 두 번째 client에 전달하면 Authorization 헤더가 자동 처리된다
- 요구사항에 따라 DB에 유저 자동 추가는 하지 않고, 기존 가입된 사용자만 로그인할 수 있다

#### 8. 환경변수 파일 업데이트 (`backend/.env`, `backend/.env.example`)
```env
# backend/.env.example에 추가
GOOGLE_CLIENT_ID=<your-google-client-id>
GOOGLE_CLIENT_SECRET=<your-google-client-secret>
GOOGLE_REDIRECT_URI=postmessage
```
- **무엇을 하는가**: 발급받은 구글 OAuth 자격증명을 프로젝트 환경변수에 등록한다
- CLIENT_SECRET은 절대 프론트엔드 환경변수나 git에 커밋하지 않는다
