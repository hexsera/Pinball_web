# 회원가입 후 자동 로그인 & 구글 버튼 흰색 배경 실행계획

## 요구사항 요약

**요구사항**:
1. 구글 로그인 버튼 배경색을 흰색으로 변경
2. 회원가입 완료 시 로그인 페이지 이동 없이 즉시 로그인 처리

**목적**: UX 개선. 구글 버튼은 흰색 배경이 표준 디자인이고, 회원가입 후 별도 로그인 없이 바로 서비스를 사용할 수 있게 한다.

## 현재상태 분석

- `auth.py`: `login`과 `google_login` 두 핸들러에서 토큰 생성 코드(`create_access_token` + `LoginResponse` 반환)가 각각 중복 작성되어 있음
- `register_user`: `UserResponse`를 반환하고 `navigate('/login')`으로 이동시킴. 토큰 미발급
- `Login.jsx`: 구글 버튼이 MUI `Button`의 인라인 `sx`로 구현. `backgroundColor` 미지정

## 구현 방법

- **토큰 생성 헬퍼**: `auth.py` 내에 `_create_login_response(user)` 헬퍼 함수를 만들어 `login`, `google_login`, `register_user` 세 곳에서 공통 사용
- **register 응답 변경**: `response_model`을 `LoginResponse`로 교체, HTTP 상태코드는 `201` 유지
- **구글 버튼**: 기존 `sx`에 `backgroundColor: '#ffffff'` 추가

## 구현 단계

### 1. 토큰 생성 헬퍼 함수 추가 (auth.py)

```python
def _create_login_response(user) -> LoginResponse:
    token = create_access_token(
        {"sub": str(user.id), "email": user.email, "role": user.role}
    )
    return LoginResponse(
        message="Login successful",
        user_id=user.id,
        email=user.email,
        nickname=user.nickname,
        role=user.role,
        access_token=token,
        token_type="bearer",
    )
```

- **무엇을 하는가**: 토큰 생성과 `LoginResponse` 조립을 한 곳으로 통일
- `login`, `google_login`, `register_user` 세 핸들러의 중복 코드를 이 함수 한 줄 호출로 교체

### 2. 기존 두 핸들러 중복 코드 교체 (auth.py)

```python
# login 핸들러 마지막 부분
return _create_login_response(user)

# google_login 핸들러 마지막 부분
return _create_login_response(user)
```

- **무엇을 하는가**: 기존 `login`, `google_login`의 token 생성 + LoginResponse 반환 코드를 헬퍼 호출 1줄로 교체

### 3. register 핸들러 응답 변경 (auth.py)

```python
# 변경 전
@router.post("/register", response_model=UserResponse, status_code=201)

# 변경 후
@router.post("/register", response_model=LoginResponse, status_code=201)

# 함수 마지막 return 교체
return _create_login_response(db_user)
```

- **무엇을 하는가**: 회원가입 성공 시 `UserResponse` 대신 `LoginResponse`(토큰 포함)를 반환
- HTTP 201 상태코드는 그대로 유지

### 4. 프론트엔드 Register.jsx 응답 처리 변경

```jsx
const { login } = useContext(AuthContext);

if (response.ok) {
  const data = await response.json();
  login({ ...data, name: data.nickname });
  navigate('/');
}
```

- **무엇을 하는가**: 회원가입 응답에서 바로 토큰을 꺼내 `login()` 호출. 추가 API 요청 없음
- 기존 `navigate('/login')` 코드를 위 코드로 교체
- `AuthContext`의 `login()`이 localStorage와 전역 상태를 동시에 업데이트

### 5. 구글 버튼 배경색 변경 (Login.jsx)

```jsx
sx={{
  width: '300px',
  backgroundColor: '#ffffff',
  borderColor: '#4285F4',
  color: '#000000',
  textTransform: 'none',
  fontSize: '0.95rem',
  '&:hover': { backgroundColor: '#f5f5f5' },
}}
```

- **무엇을 하는가**: 기존 `sx`에 `backgroundColor: '#ffffff'` 추가, 텍스트 색상을 흰 배경에 맞게 검정으로 변경
