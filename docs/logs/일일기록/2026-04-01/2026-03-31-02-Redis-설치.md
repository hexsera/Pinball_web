## 작업 개요

게임 세션 유지 기능에서 사용할 Redis를 기존 Docker Compose 스택에 추가한다.  
FastAPI 백엔드에서 Redis에 접근할 수 있도록 연결 클라이언트도 함께 설정한다.

---

## 현재 환경

- Docker Compose로 traefik, nginx, fastapi, postgres, mysql 서비스 운영 중
- 모든 컨테이너는 `web` 네트워크 사용
- FastAPI 의존 패키지: `requirements.txt` 관리

---

## 구현 계획

### 1. docker-compose.yml — Redis 서비스 추가

```yaml
redis:
  image: redis:8.6.2
  container_name: redis-server
  restart: always
  ports:
    - "6379:6379"
  volumes:
    - redis-data:/data
  networks:
    - web
```

- volumes 섹션에 `redis-data:` 추가
- fastapi 서비스의 `depends_on`에 `redis` 추가

### 2. backend/requirements.txt — redis 패키지 추가

```
redis==7.1.1
```

### 3. backend/app/redis_client.py — 신규 생성

Redis 연결 클라이언트 모듈.

```python
import redis
import os

redis_client = redis.Redis(
    host=os.getenv("REDIS_HOST", "redis-server"),
    port=int(os.getenv("REDIS_PORT", 6379)),
    decode_responses=True
)
```

- `decode_responses=True`: 값을 bytes 대신 str로 반환
- 호스트명 `redis-server`는 docker-compose 컨테이너명과 일치

### 4. backend/.env — Redis 환경 변수 추가 (선택)

```
REDIS_HOST=redis-server
REDIS_PORT=6379
```

`.env.example`에도 동일하게 추가.

## 테스트 조건

설치 완료 후 아래 조건을 순서대로 확인한다. 모든 항목이 통과해야 정상 설치로 간주한다.

### 조건 1. Redis 컨테이너가 정상 기동되었는가

```bash
docker compose up -d redis
docker compose ps
```

- `redis-server` 상태가 `Up`이어야 한다.

---

### 조건 2. Redis가 요청에 응답하는가

```bash
docker compose exec redis redis-cli ping
```

- 응답이 `PONG`이어야 한다.

---

### 조건 3. Redis에 데이터 쓰기/읽기가 되는가

```bash
docker compose exec redis redis-cli set test_key "hello"
docker compose exec redis redis-cli get test_key
```

- `set` 응답이 `OK`이어야 한다.
- `get` 응답이 `"hello"`이어야 한다.

---

### 조건 4. FastAPI 컨테이너에서 Redis에 연결되는가

```bash
docker compose exec fastapi python -c "from app.redis_client import redis_client; print(redis_client.ping())"
```

- 응답이 `True`이어야 한다.

---

### 조건 5. FastAPI에서 Redis 데이터 쓰기/읽기가 되는가

```bash
docker compose exec fastapi python -c "
from app.redis_client import redis_client
redis_client.set('test_key', 'hello', ex=10)
print(redis_client.get('test_key'))
"
```

- 응답이 `hello`이어야 한다.
- `ex=10`은 10초 TTL 설정 — 만료 기능도 함께 검증한다.
