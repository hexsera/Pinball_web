# 게임 세션 유지 — 프론트엔드 실행계획

## 요구사항 요약

**요구사항**: 로그인 사용자가 핀볼 게임 도중 페이지를 이탈했다가 돌아올 때 이전 게임 상태(점수·목숨·스테이지)를 자동 복원한다.

**목적**: 인터넷 끊김, 기기 전환 등 의도치 않은 이탈 후 게임을 처음부터 다시 시작하지 않도록 한다.

## 현재상태 분석

- `Pinball.jsx`: score/lives/stage를 useState + useRef 이중으로 관리. 마운트 시 항상 초기값(0, 3, 1)으로 시작.
- `handleRestart()`: 항상 초기값으로 리셋. 세션 삭제 로직 없음.
- `submitScore()`: 게임오버 시 점수 제출. 세션 삭제 로직 없음.
- `deathZone` 충돌 처리: 목숨 감소 후 게임오버 판별. 세션 저장 로직 없음.
- `frontend/src/utils/gameSession.js`: 미존재.

## 구현 방법

- `gameSession.js` 유틸 파일을 생성해 API 호출을 캡슐화한다.
- `Pinball.jsx`에 useEffect 2개(마운트 시 복원, 주기적 저장)를 추가하고, 기존 deathZone/submitScore/handleRestart에 세션 저장·삭제 호출을 추가한다.
- 백엔드 API: GET/PUT/DELETE `/api/v1/game-sessions/{user_id}` (백엔드 구현 전이므로 실제 동작 확인은 백엔드 완료 후 수행).

## 구현 단계

### 1. gameSession.js 생성

```javascript
// frontend/src/utils/gameSession.js
import axios from 'axios';

export async function saveGameSession(userId, { score, lives, stage }) {
  await axios.put(`/api/v1/game-sessions/${userId}`, { score, lives, stage });
}

export async function loadGameSession(userId) {
  try {
    const res = await axios.get(`/api/v1/game-sessions/${userId}`);
    return res.data;
  } catch (e) {
    if (e.response?.status === 404) return null;
    throw e;
  }
}

export async function deleteGameSession(userId) {
  await axios.delete(`/api/v1/game-sessions/${userId}`);
}
```

- 404는 세션 없음(정상)으로 처리해 null 반환, 그 외 오류는 호출부로 전파.

### 2. 마운트 시 세션 복원 useEffect 추가

```javascript
// Pinball.jsx — game_visits useEffect 아래에 추가
useEffect(() => {
  if (!user?.id) return;
  loadGameSession(user.id).then(session => {
    if (!session) return;
    setScore(session.score);
    setLives(session.lives);
    setStage(session.stage);
    scoreRef.current = session.score;
    livesRef.current = session.lives;
    stageRef.current = session.stage;
  }).catch(err => console.error('세션 복원 실패:', err));
}, []);
```

- 비로그인이거나 세션 없으면 초기값(0, 3, 1) 유지.
- 복원 후에도 게임 시작 화면(Space/클릭 대기)은 그대로 유지됨 — `gameStarted` state를 건드리지 않으므로 별도 처리 불필요.

### 3. 주기적 저장 useEffect 추가

```javascript
// Pinball.jsx — 마운트 세션 복원 useEffect 아래에 추가
useEffect(() => {
  if (!user?.id) return;
  const interval = setInterval(() => {
    saveGameSession(user.id, {
      score: scoreRef.current,
      lives: livesRef.current,
      stage: stageRef.current,
    }).catch(err => console.error('세션 저장 실패:', err));
  }, 30000);
  return () => clearInterval(interval);
}, []);
```

- ref 값을 직접 읽으므로 React 렌더링 주기와 무관하게 항상 최신 값 저장.
- 컴포넌트 언마운트 시 interval 정리.

### 4. 목숨 감소 시 즉시 저장

```javascript
// Pinball.jsx — deathZone 충돌 처리 내 livesRef.current = newLives; 직후
if (user?.id) {
  saveGameSession(user.id, {
    score: scoreRef.current,
    lives: newLives,
    stage: stageRef.current,
  }).catch(err => console.error('세션 저장 실패:', err));
}
```

- 목숨 감소는 가장 중요한 상태 변화이므로 30초 주기를 기다리지 않고 즉시 저장.

### 5. 게임오버 시 세션 삭제

```javascript
// Pinball.jsx — submitScore() 함수 내 try 블록 마지막에 추가
if (user?.id) {
  await deleteGameSession(user.id)
    .catch(err => console.error('세션 삭제 실패:', err));
}
```

- 점수 제출 성공 여부와 무관하게 세션은 삭제한다. (catch 안에도 추가 불필요 — 게임오버 시 어차피 새 게임 시작)

### 6. handleRestart() 세션 삭제 추가

```javascript
// Pinball.jsx — handleRestart() 함수 내 초기화 로직 마지막에 추가
if (user?.id) {
  deleteGameSession(user.id)
    .catch(err => console.error('세션 삭제 실패:', err));
}
```

- 재시작은 의도적인 새 게임 시작이므로 이전 세션을 삭제한다.
