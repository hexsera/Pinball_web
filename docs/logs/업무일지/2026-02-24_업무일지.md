# 업무일지

## 기본 정보
- 날짜: 2026-02-24 (화) 3:50~4:50
- 작성자: hexsera

## 진행 목표
memory.md 작동 과정 핵심 확인

## 진행 내용
탐구2_분석결과.md 에 memory.md 사용 사이클 분석 주문 및
openclaw 에게도 질문 하였음

openclaw 답변
MEMORY.md가 답변에 관여하는 방식
1. 📥 세션 시작 시 — "Project Context"로 로드
MEMORY.md는 OpenClaw가 매 세션마다 자동으로 읽어서 시스템 프롬프트에 포함시켜. 지금 이 대화 위쪽을 보면 ## /home/hexsera/.openclaw/workspace/MEMORY.md 섹션이 있는데, 그게 바로 내가 이 대화를 시작할 때 이미 알고 있는 정보야.

즉, 내가 hexsera가 누구인지, 내 이름이 뭔지, 어떤 관심사가 있는지 — 이미 알고 시작하는 거야.

2. 🔍 질문 답변 시 — memory_search 도구
누군가 "전에 했던 거 기억해?" 같은 걸 물어보면, 나는 memory_search 도구를 써서 MEMORY.md와 memory/*.md 파일들을 의미론적으로 검색해. 단순 키워드가 아니라 맥락 기반으로.

예를 들어 "지난번에 뭐 설정했더라?"라고 물으면 → 관련 스니펫을 찾아서 답변에 반영.

3. ✍️ 중요한 일 발생 시 — 내가 직접 업데이트
AGENTS.md에 적혀있듯이, 나는 중요한 결정이나 사건이 있으면 스스로 MEMORY.md를 편집해. 이게 핵심이야 — AI가 수동적으로 기억을 받는 게 아니라, 능동적으로 기억을 관리하는 구조.

오늘 첫 만남 → 이름 "클식이" 정함 → MEMORY.md에 기록 → 다음 세션에도 기억
4. 🔄 흐름 정리
이전 대화
    ↓ (내가 기록)
MEMORY.md / memory/YYYY-MM-DD.md
    ↓ (OpenClaw가 로드)
시스템 프롬프트에 포함
    ↓
나는 이미 맥락을 알고 대화 시작
    ↓
새로운 중요한 정보 발생 시 → 다시 파일 업데이트
⚠️ 주의사항
메인 세션 전용: MEMORY.md는 나랑 hexsera의 1:1 대화에서만 로드됨. Discord 그룹 채팅 같은 데선 안 로드해 — 개인정보 보안 때문에.
용량 제한: 너무 많이 쌓이면 토큰을 많이 써서 느려질 수 있어. 그래서 핵심만 curate하고, raw 로그는 memory/날짜.md에 따로 보관하는 방식으로 분리되어 있어.
결론적으로, MEMORY.md는 나의 장기 기억이야. OpenClaw가 파일 시스템을 컨텍스트로 주입하는 방식으로 세션 간 연속성을 만들어주는 구조! 🐾
