# 업무일지

## 기본 정보
- 날짜: 2026-02-26 (목) 19:00~20:30
- 작성자: hexsera

## 진행 목표
임베딩 저장 분석 및 정리

## 진행 내용
1. 파일 청킹:
memory.py 의 파일을 정해진 토큰(글자수) 만큼 자른다.
2. 청킹 캐시:
청크를 해쉬함수로 해쉬화 하여
임베딩이 저장된 DB 에 동일한 해쉬가 있는지 확인하여 동일한 해쉬가 있다면 해당 청크 임베딩 생략.
3. 임베딩 api:
api 호출하여 모델, 텍스트(청크), tasktype 을 지정해준다.
4. 벡터 정규화:
벡터를 정규화 시킨다. 단 gemini 와 같은 상용 api 는 정규화 값이 나온다고함.
5. DB 저장:
해당 청크를 [청크파일경로, 청크 텍스트, 청크 해쉬, 청크 벡터] 로 DB 에 저장한다.
