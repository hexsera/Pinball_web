```jsx
import React, { useState, useEffect } from 'react';
import {
  Box, Paper, TextField, Button, List, ListItem,
  ListItemText, Typography, Alert, Divider, Toolbar, Container,
```

**변경 의도:**

JWT 인증 개선 작업에서 `login()` 시 `localStorage.setItem('user', ...)` 호출이 제거됐다. 그 결과 `localStorage.getItem('user')`는 항상 `null`을 반환하고, `fetchFriendList` / `fetchPendingRequests` 모두 API 요청 없이 조기 반환됐다. 실제 유저 정보는 `AuthContext`(`useAuth`)가 메모리에서 관리하므로, `getCurrentUser()` 헬퍼를 완전히 제거하고 `useAuth().user`로 교체했다.

`useEffect` 의존성 배열에 `currentUser`를 추가한 이유: `AuthContext`는 마운트 시 `/auth/refresh`를 비동기로 호출한 뒤 `user`를 세팅하므로, 컴포넌트 첫 렌더링 시 `currentUser`가 `null`일 수 있다. 의존성으로 등록해야 `user` 세팅 완료 후 fetch가 재실행된다.
