# 구글 로그인 실행계획

## 처리 흐름도

```
[사용자]
   │
   │ 1. "구글로 로그인" 버튼 클릭
   ▼
[Frontend: @react-oauth/google]
   │
   │ 2. 구글 계정 선택 팝업 열림
   ▼
[Google 서버]
   │
   │ 3. Authorization Code 발급
   ▼
[Frontend]
   │
   │ 4. POST /api/v1/auth/google  { code: "..." }
   ▼
[Backend: FastAPI]
   │
   │ 5. POST https://oauth2.googleapis.com/token
   │    { code, client_id, client_secret, redirect_uri, grant_type }
   ▼
[Google 서버]
   │
   │ 6. Access Token 반환
   ▼
[Backend: FastAPI]
   │
   │ 7. GET https://www.googleapis.com/oauth2/v3/userinfo
   │    Authorization: Bearer {access_token}
   ▼
[Google 서버]
   │
   │ 8. { sub, email, name } 반환
   ▼
[Backend: FastAPI]
   │
   │ 9. DB에서 email 조회 → 없으면 자동 생성
   ▼
[PostgreSQL]
   │
   │ 10. { user_id, email, nickname, role } 반환
   ▼
[Backend: FastAPI]
   │
   │ 11. LoginResponse 반환
   ▼
[Frontend]
   │
   │ 12. localStorage에 사용자 정보 저장 → 홈(/) 이동
   ▼
[사용자]
```

> Access Token은 5~8번 구간(서버-서버)에서만 존재. 브라우저에 노출되지 않음.

---

## 요구사항 요약

**요구사항**: Google OAuth 2.0 Authorization Code Flow를 사용하여 구글 계정으로 로그인/회원가입 기능 추가

**목적**: 이메일/비밀번호 입력 없이 구글 계정으로 간편하게 로그인하여 사용자 진입 장벽을 낮춘다.

## 현재상태 분석

- **인증 방식**: `POST /api/v1/login`에 email + password 전송 → 백엔드 bcrypt 검증 → localStorage에 `{id, name, role, email}` 저장
- **세션 관리**: JWT 없음, localStorage 기반 평문 저장
- **User 테이블**: `password` 컬럼 있음, `user_id` UUID 컬럼 있음 (nullable)
- **현재 OAuth 패키지**: 없음 (frontend: react 19, backend: fastapi 0.109)

## 구현 방법

- **Frontend**: `@react-oauth/google` (2026 기준 React용 Google OAuth 표준 패키지)
- **Backend**: `Authlib 1.6.9` (2026 기준 FastAPI OAuth 2.0 표준 라이브러리) + `httpx` (비동기 HTTP 클라이언트)
- **Authlib 활용 범위**: 팝업 방식이므로 Authlib의 리다이렉트 흐름은 사용하지 않고, `AsyncOAuth2Client`로 Token 교환 및 userinfo 조회에 활용
- **흐름**:
  1. 프론트엔드: 구글 로그인 팝업 → 구글이 Authorization Code 발급 → 백엔드로 전송
  2. 백엔드: Authlib `AsyncOAuth2Client`로 Authorization Code → Token 교환 → userinfo 조회
  3. 백엔드: DB에 유저 없으면 자동 생성 → 기존 로그인 응답과 동일한 형태 반환
- **보안**: Access Token이 브라우저에 노출되지 않음 (구글 권장 방식)

## 구현 단계

### Phase 1: Frontend

#### 1. 패키지 설치
```bash
npm install @react-oauth/google
```
- **무엇을 하는가**: React 앱에서 Google OAuth 2.0을 처리하는 공식 패키지 설치
- `@react-oauth/google`은 Google Identity Services 기반으로 2022년 이후 표준

#### 2. GoogleOAuthProvider 설정 (`frontend/src/main.jsx`)
```jsx
import { GoogleOAuthProvider } from '@react-oauth/google';

ReactDOM.createRoot(document.getElementById('root')).render(
  <GoogleOAuthProvider clientId={import.meta.env.VITE_GOOGLE_CLIENT_ID}>
    <App />
  </GoogleOAuthProvider>
);
```
- **무엇을 하는가**: 앱 전체에서 Google OAuth를 사용할 수 있도록 최상위에 Provider 래핑
- `VITE_GOOGLE_CLIENT_ID`: Google Cloud Console에서 발급받은 OAuth 2.0 클라이언트 ID

#### 3. Google 로그인 버튼 추가 (`frontend/src/pages/Login/Login.jsx`)
```jsx
import { useGoogleLogin } from '@react-oauth/google';

const Login = () => {
  const googleLogin = useGoogleLogin({
    flow: 'auth-code',  // Authorization Code Flow
    onSuccess: async (codeResponse) => {
      // codeResponse.code = Authorization Code (Access Token 아님)
      const res = await axios.post('/api/v1/auth/google', {
        code: codeResponse.code,
      });
      if (res.status === 200) {
        login({ id: res.data.user_id, name: res.data.nickname, role: res.data.role, email: res.data.email });
        navigate(res.data.role === 'admin' ? '/admin' : '/');
      }
    },
    onError: () => alert('구글 로그인에 실패했습니다.'),
  });

  return (
    // ... 기존 폼 ...
    <button onClick={() => googleLogin()}>구글로 로그인</button>
  );
};
```
- **무엇을 하는가**: 구글 로그인 팝업 → Authorization Code만 받아 백엔드로 전송 (Access Token은 브라우저에 오지 않음)
- `flow: 'auth-code'` 설정이 핵심 — 이것이 없으면 Implicit Flow로 동작
- 성공 시 기존 `login()` 함수 재사용하여 localStorage 저장 및 리다이렉트

#### 4. 환경변수 추가 (`frontend/.env`)
```env
VITE_GOOGLE_CLIENT_ID=your_google_client_id_here
```
- **무엇을 하는가**: Google Cloud Console에서 발급받은 클라이언트 ID를 환경변수로 관리
- Vite는 `VITE_` 접두사가 있는 변수만 클라이언트에 노출
- CLIENT_SECRET은 프론트엔드에 절대 두지 않음 (백엔드 전용)

### Phase 2: Backend

#### 5. 패키지 설치 (`backend/requirements.txt`)
```txt
Authlib==1.6.9
httpx==0.27.0
```
- **무엇을 하는가**: Authlib는 OAuth 2.0 Token 교환 및 userinfo 조회를 처리하는 라이브러리, httpx는 Authlib의 비동기 HTTP 백엔드로 함께 필요

#### 6. 환경변수 추가 (`backend/.env`, `backend/app/core/config.py`)
```python
# config.py
GOOGLE_CLIENT_ID: str = os.getenv("GOOGLE_CLIENT_ID", "")
GOOGLE_CLIENT_SECRET: str = os.getenv("GOOGLE_CLIENT_SECRET", "")
GOOGLE_REDIRECT_URI: str = os.getenv("GOOGLE_REDIRECT_URI", "postmessage")
```
```env
# .env
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GOOGLE_REDIRECT_URI=postmessage
```
- **무엇을 하는가**: Authorization Code → Token 교환 시 구글이 CLIENT_SECRET으로 백엔드를 검증
- `GOOGLE_REDIRECT_URI=postmessage`: 팝업 방식 로그인 시 구글이 요구하는 고정값

#### 7. Google OAuth 엔드포인트 생성 (`backend/app/api/v1/auth.py`)
```python
from authlib.integrations.httpx_client import AsyncOAuth2Client

@router.post("/auth/google")
async def google_login(data: GoogleLoginRequest, db: Session = Depends(get_db)):
    # 1. Authlib AsyncOAuth2Client로 Authorization Code → Token 교환
    async with AsyncOAuth2Client(
        client_id=settings.GOOGLE_CLIENT_ID,
        client_secret=settings.GOOGLE_CLIENT_SECRET,
    ) as client:
        token = await client.fetch_token(
            "https://oauth2.googleapis.com/token",
            code=data.code,
            redirect_uri=settings.GOOGLE_REDIRECT_URI,
            grant_type="authorization_code",
        )

    # 2. 발급된 token으로 구글 userinfo 조회
    async with AsyncOAuth2Client(token=token) as client:
        resp = await client.get("https://www.googleapis.com/oauth2/v3/userinfo")

    if resp.status_code != 200:
        raise HTTPException(status_code=401, detail="Failed to get Google user info")

    google_user = resp.json()  # {"sub": "...", "email": "...", "name": "..."}

    # 3. DB에서 이메일로 사용자 조회, 없으면 자동 생성
    user = db.query(User).filter(User.email == google_user["email"]).first()
    if not user:
        user = User(
            email=google_user["email"],
            nickname=google_user.get("name", google_user["email"].split("@")[0]),
            password=None,
            birth_date=None,
            role="user"
        )
        db.add(user)
        db.commit()
        db.refresh(user)

    return LoginResponse(
        message="Login successful",
        user_id=user.id,
        email=user.email,
        nickname=user.nickname,
        role=user.role
    )
```
- **무엇을 하는가**: Authlib `AsyncOAuth2Client`가 Token 교환 요청을 구성하고 처리 — 직접 HTTP 요청을 구성할 필요 없음
- `fetch_token()`이 `client_id`, `client_secret`, `code`를 조합해 구글 token 엔드포인트에 POST 전송
- token 객체를 그대로 두 번째 client에 전달하면 Authorization 헤더 자동 처리

#### 8. Pydantic 스키마 추가 (`backend/app/schemas/user.py`)
```python
class GoogleLoginRequest(BaseModel):
    code: str  # Authorization Code
```
- **무엇을 하는가**: 프론트엔드에서 전송하는 Authorization Code를 받는 요청 스키마
- 기존 `access_token: str`에서 `code: str`로 변경

#### 9. User 모델 password nullable 처리 (`backend/models.py`)
```python
password: Mapped[Optional[str]] = mapped_column(String(255), nullable=True)
birth_date: Mapped[Optional[date]] = mapped_column(Date, nullable=True)
```
- **무엇을 하는가**: 구글 로그인 사용자는 비밀번호가 없으므로 password 컬럼을 nullable로 변경
- 기존 이메일/비밀번호 로그인 사용자에게는 영향 없음

#### 10. DB 마이그레이션 실행
```bash
docker compose exec fastapi alembic revision --autogenerate -m "make password nullable for oauth"
docker compose exec fastapi alembic upgrade head
```
- **무엇을 하는가**: password 컬럼의 nullable 변경사항을 실제 PostgreSQL DB에 적용

### Phase 3: Google Cloud Console 등록

#### 11. Google Cloud 프로젝트 생성
```
https://console.cloud.google.com/
```
- **무엇을 하는가**: OAuth 클라이언트 ID를 발급받기 위한 Google Cloud 프로젝트 생성
- 좌측 상단 프로젝트 드롭다운 → **NEW PROJECT** → 프로젝트 이름 입력 → **CREATE**

#### 12. OAuth 동의 화면(Consent Screen) 설정
```
APIs & Services → OAuth consent screen
```
- **무엇을 하는가**: 사용자가 구글 로그인 팝업에서 보게 될 앱 정보와 요청 권한을 등록
- **User Type**: **External** 선택 (일반 Google 계정 허용)
- **필수 입력**: App name, User support email, Developer contact email
- **Scopes 추가**: `userinfo.profile`, `userinfo.email` 두 가지만 추가
- **2026년 주의**: 사용자가 개별 권한을 거부할 수 있는 Granular Consent 적용됨

#### 13. 테스트 사용자 추가
```
APIs & Services → OAuth consent screen → TEST USERS → ADD USERS
```
- **무엇을 하는가**: 앱이 "게시 전" 상태일 때 로그인 가능한 Google 계정을 등록
- External 앱은 게시 전까지 테스트 사용자(최대 100명)만 로그인 가능
- 등록되지 않은 계정으로 로그인 시 `oauth_unauthenticated_user` 오류 발생

#### 14. OAuth 2.0 클라이언트 ID 생성
```
APIs & Services → Credentials → CREATE CREDENTIALS → OAuth client ID
```
- **무엇을 하는가**: 프론트엔드/백엔드에서 사용할 CLIENT_ID, CLIENT_SECRET 발급
- **Application type**: **Web application** 선택
- **Authorized JavaScript origins** 등록 (postmessage 방식이므로 redirect URI 불필요):
  ```
  # 개발
  http://localhost:5173

  # 프로덕션
  https://your-domain.com
  ```
- **CREATE** 후 표시되는 CLIENT_ID, CLIENT_SECRET을 즉시 복사하여 저장

#### 15. 환경변수 파일에 발급받은 값 입력
```bash
# backend/.env
GOOGLE_CLIENT_ID=123456789-xxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-xxxxxxxxxx
GOOGLE_REDIRECT_URI=postmessage

# frontend/.env
VITE_GOOGLE_CLIENT_ID=123456789-xxxx.apps.googleusercontent.com
```
- **무엇을 하는가**: 발급받은 자격증명을 프로젝트 환경변수에 등록
- `GOOGLE_REDIRECT_URI=postmessage`는 팝업 방식 고정값으로 Google Cloud Console에 별도 등록 불필요
- CLIENT_SECRET은 절대 프론트엔드 환경변수나 git에 커밋하지 않음
