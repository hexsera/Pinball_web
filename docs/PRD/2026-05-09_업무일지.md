## 진행 목표

어드민 페이지(`/admin`, `/admin/users`, `/admin/statistics`)에 프론트엔드 라우트 가드를 도입하여, `role=admin`인 사용자만 진입 가능하도록 한다. 비로그인 및 일반 user는 `/`로 리다이렉트한다.

## 주요 변경 사항

### 1. frontend/src/components/AdminRoute.jsx 생성

**추가:**
```jsx
import { Navigate, Outlet } from 'react-router-dom';
import { jwtDecode } from 'jwt-decode';
import { useAuthStore } from '../store/authStore';

export default function AdminRoute() {
  const accessToken = useAuthStore((s) => s.accessToken);
  if (!accessToken) return <Navigate to="/" replace />;

  try {
    const { role } = jwtDecode(accessToken);
    if (role !== 'admin') return <Navigate to="/" replace />;
  } catch {
    return <Navigate to="/" replace />;
  }

  return <Outlet />;
}
```

**변경 의도:** 어드민 라우트 진입 시 JWT payload를 디코딩해 `role === 'admin'`인 경우에만 자식 라우트를 렌더링한다. 비로그인·일반 user·디코딩 실패 모두 `/`로 통일 리다이렉트. 서명 검증은 하지 않으며(백엔드 책임), payload 디코딩만 수행한다.

**변경 후:**
```jsx
import { AdminUserPage, AdminStatisticsPage } from './pages/admin';
import AdminRoute from './components/AdminRoute';

<Route element={<AdminRoute />}>
  <Route path="/admin" element={<AdminUserPage/>} />
  <Route path="/admin/users" element={<AdminUserPage/>} />

```

**변경 의도:** 어드민 라우트 3개를 Outlet 패턴으로 묶어 `AdminRoute` 가드 한 번으로 일괄 보호. `/admin`과 `/admin/users` 중복 매핑은 기존 의도를 유지하여 그대로 둔다(후속 검토 예정).

### 3. frontend/src/test/AdminRoute.test.jsx 생성

**추가:**
```jsx
import { render, screen } from '@testing-library/react';
import { MemoryRouter, Routes, Route } from 'react-router-dom';
import { describe, it, expect, vi, beforeEach } from 'vitest';
import AdminRoute from '../components/AdminRoute';
import { useAuthStore } from '../store/authStore';

vi.mock('jwt-decode', () => ({
  jwtDecode: vi.fn(),
}));

import { jwtDecode } from 'jwt-decode';

function setup(token, role) {
  useAuthStore.setState({ accessToken: token });

  if (token && role !== undefined) {
    jwtDecode.mockReturnValue({ role });
  } else if (token && role === undefined) {
    jwtDecode.mockImplementation(() => { throw new Error('invalid token'); });
  }

  render(
    <MemoryRouter initialEntries={['/admin']}>
      <Routes>
        <Route element={<AdminRoute />}>
          <Route path="/admin" element={<div>Admin Page</div>} />
        </Route>
        <Route path="/" element={<div>Home</div>} />
      </Routes>
    </MemoryRouter>
  );
}

describe('AdminRoute', () => {
  beforeEach(() => {
    vi.clearAllMocks();
    useAuthStore.setState({ accessToken: null });
  });

  it('토큰 없으면 Home으로 리다이렉트', () => {
    setup(null, null);
    expect(screen.getByText('Home')).toBeInTheDocument();
  });

  it('role=admin이면 어드민 페이지 렌더', () => {
    setup('valid.token.here', 'admin');
    expect(screen.getByText('Admin Page')).toBeInTheDocument();
  });

  it('role=user면 Home으로 리다이렉트', () => {
    setup('valid.token.here', 'user');
    expect(screen.getByText('Home')).toBeInTheDocument();
  });

  it('디코딩 실패하면 Home으로 리다이렉트', () => {
    setup('broken.token', undefined);
    expect(screen.getByText('Home')).toBeInTheDocument();
  });
});
```

**변경 의도:** `AdminRoute` 4가지 분기(토큰 없음, admin, user, 디코딩 실패)를 단위 테스트로 검증. `jwtDecode`와 `useAuthStore`를 모킹해 실제 JWT 없이 테스트 가능하도록 구성.

## 라우트 가드 동작 흐름

```
브라우저 → /admin 접속
              │
              ▼
        AdminRoute 실행
              │
     accessToken 있음?
       │            │
      No            Yes
       │            │
       ▼            ▼
  → / 리다이렉트  jwtDecode(token)
                    │
              role === 'admin'?
               │           │
              No            Yes
               │            │
               ▼            ▼
          → / 리다이렉트  <Outlet /> 렌더
                            │
                            ▼
                     자식 라우트 표시
                  (AdminUserPage 등)
```
