## 변경 내용

### 1. localStorage 제거

- **변경 전**: Redis(주) + localStorage(fallback)
- **변경 후**: Redis만 사용
- **이유**: Redis 장애 시 FastAPI도 불안정하여 fallback 실효성 낮음. 구현 복잡도만 증가하므로 제거.

### 2. 세션 복원 시나리오 확정

- **대상 시나리오**: 인터넷 끊김, 기기 전환 등 의도치 않은 이탈 후 복귀
- **방식**: 자동 복원 (사용자에게 묻지 않음)
- **이유**: 해당 시나리오는 사용자가 게임을 계속하길 원하는 상황. 다이얼로그는 불필요한 마찰.
- **주의**: 복원 후에도 게임 시작 화면(Space/클릭 대기)은 유지. 세션 없거나 비로그인이면 초기값(0, 3, 1).

### 3. Redis TTL 변경

- **변경 전**: 24시간
- **변경 후**: 2시간
- **이유**: 복구 시나리오(인터넷 끊김·기기 변경)는 수 분~수 시간 내 복귀가 대부분. 24시간은 과도하며 오래된 세션이 의도치 않게 복원될 수 있음.

### 4. GET 응답 방식 변경

- **변경 전**: 세션 없으면 `null` 반환 (404 아님)
- **변경 후**: 세션 없으면 `404` 반환
- **이유**: 프로젝트 전체가 RESTful 스타일로 작성됨. 일관성 유지. 프론트엔드 `loadGameSession`에서 404를 null로 변환하여 처리.
