## 1. React Query 란?

React Query(`@tanstack/react-query`)는 서버에서 데이터를 가져오고 캐싱하는 작업을 대신 처리해주는 라이브러리다.

React에서 API 데이터를 가져올 때 보통 이렇게 한다:

```jsx
const [data, setData] = useState(null);
const [loading, setLoading] = useState(false);

useEffect(() => {
  setLoading(true);
  axios.get('/api/data').then(res => {
    setData(res.data);
    setLoading(false);
  });
}, []);
```

React Query를 쓰면 이 보일러플레이트가 사라진다:

```jsx
const { data, isPending } = useQuery({
  queryKey: ['data'],
  queryFn: () => axios.get('/api/data').then(res => res.data),
});
```

---

## 2. React Query 를 왜 사용하는가?

`useEffect` + `useState` 방식의 문제점:

| 문제 | 설명 |
|------|------|
| 로딩/에러 상태를 직접 관리 | `useState`로 `loading`, `error`를 매번 선언해야 함 |
| 캐시 없음 | 같은 데이터를 요청할 때마다 매번 API 호출 |
| 중복 요청 | 여러 컴포넌트가 같은 API를 동시에 호출하면 요청이 중복 발생 |
| 자동 갱신 없음 | 탭 포커스, 네트워크 재연결 시 수동으로 재호출해야 함 |

React Query가 해결해주는 것:

- **로딩/에러 상태 자동 제공** — `isPending`, `isError` 자동으로 관리
- **캐시** — 같은 `queryKey`라면 일정 시간 동안 재요청 없이 캐시 재사용
- **중복 요청 제거** — 같은 `queryKey`의 동시 요청을 하나로 합침
- **자동 리페치** — 탭 포커스, 네트워크 재연결, 일정 주기마다 자동 갱신

## 3. 설치 및 초기 셋팅

### 설치

```bash
source ~/.nvm/nvm.sh && cd frontend && npm install @tanstack/react-query
```

### QueryClient 생성 및 앱 감싸기

`main.jsx`에서 `QueryClient`를 만들고 앱 전체를 `QueryClientProvider`로 감싼다.  
이 작업을 해야 모든 컴포넌트에서 `useQuery`를 사용할 수 있다.

```jsx
// frontend/src/main.jsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

const queryClient = new QueryClient();  // 캐시 저장소 역할, 한 번만 생성

root.render(
  <QueryClientProvider client={queryClient}>
    <App />
  </QueryClientProvider>
);
```

- `QueryClient`: React Query의 캐시를 관리하는 객체
- `QueryClientProvider`: 하위 컴포넌트 전체에 `queryClient`를 전달하는 컨텍스트 Provider
- `queryClient` 인스턴스는 컴포넌트 밖에서 한 번만 생성해야 한다 (리렌더링마다 재생성 방지)

## 4. 이 프로젝트에서 사용된 React Query 기능

홈페이지 월간 랭킹에 적용된 기능 기준으로 설명한다.

### useQuery

서버에서 데이터를 가져오는 기본 훅.

```jsx
import { useQuery } from '@tanstack/react-query';

const { data, isPending, isError } = useQuery({
  queryKey: ['monthly-scores'],
  queryFn: () => axios.get('/api/v1/monthly-scores').then(res => res.data.scores.slice(0, 10)),
  staleTime: 1000 * 60 * 2,
  refetchInterval: 1000 * 30,
});
```

#### queryKey

캐시를 식별하는 키. 배열 형태로 작성한다.

```jsx
queryKey: ['monthly-scores']
```

- 같은 `queryKey`를 가진 `useQuery`는 캐시를 공유한다
- `queryKey`가 바뀌면 새 요청을 보낸다 (페이지네이션에서 활용)
- 예: `['users', page]` — page가 바뀌면 새 요청

#### queryFn

실제 API를 호출하는 함수. Promise를 반환해야 한다.

```jsx
queryFn: () => axios.get('/api/v1/monthly-scores').then(res => res.data.scores.slice(0, 10))
```

- `axios.get()`은 Promise를 반환하므로 그대로 사용 가능
- `.then()`으로 필요한 데이터만 추출해서 반환하면 `data`에 그 값이 담긴다

#### staleTime

캐시를 "신선하다"고 판단하는 시간 (밀리초).

```jsx
staleTime: 1000 * 60 * 2  // 2분
```

- 이 시간 안에 같은 `queryKey`로 다시 접근하면 API 요청 없이 캐시를 그대로 반환
- 기본값은 `0` — 매번 재요청
- 예: 홈페이지를 나갔다 2분 안에 돌아오면 Network 탭에 재요청이 없다

#### refetchInterval

일정 주기마다 자동으로 API를 재요청하는 옵션 (밀리초).

```jsx
refetchInterval: 1000 * 30  // 30초
```

- 컴포넌트가 마운트된 동안 30초마다 자동으로 `queryFn`을 실행
- Network 탭에서 주기적인 요청으로 동작 확인 가능
- 실시간성이 필요한 데이터에 적합 (채팅 목록, 알림 등)

### isPending / isError

`useQuery`가 자동으로 제공하는 상태값.

```jsx
const { data, isPending, isError } = useQuery({ ... });

{isPending ? (
  <Typography>랭킹을 불러오는 중...</Typography>
) : isError ? (
  <Typography>랭킹을 불러올 수 없습니다.</Typography>
) : (
  <TableContainer>...</TableContainer>
)}
```

| 상태값 | 설명 |
|--------|------|
| `isPending` | 최초 데이터 로딩 중일 때 `true` |
| `isError` | 요청 실패 시 `true` |
| `data` | 요청 성공 시 `queryFn`이 반환한 값 |

- 기존의 `useState(loading)`, `useState(error)` 를 직접 관리할 필요가 없어진다
