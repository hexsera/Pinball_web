## 개요

AIPinball 컴포넌트(`frontend/src/pages/AIPinball/`)의 구조를 Claude와 대화하며 점진적으로 개편하는 작업.

## 개편 전 현재 구조 (스냅샷)

### 파일 목록

| 파일 | 역할 |
|------|------|
| `AIPinball.jsx` | 메인 컴포넌트. 플레이어+AI 플리퍼, 플레이스타일 분석, 필살기 로직 전부 포함 |
| `aiStageConfigs.js` | 스테이지별 범퍼/장애물/목표물 좌표 데이터 |
| `playstyleService.js` | 플레이스타일 API 통신 + 응답 파싱 + 랜덤 폴백 |
| `SkillIcon.jsx` | 스킬 상태 표시 아이콘 UI (`loading` / `big` / `small`) |
| `index.js` | AIPinball 컴포넌트 re-export |

Pinball 폴더에서 공유:
- `../Pinball/pinballSound` — 사운드 재생 유틸
- `../Pinball/pinballRestart` — 재시작 초기 상태값
- `../Pinball/WallOverlay` — CSS 벽 오버레이

### 물리 오브젝트 레이아웃

```
x=0  40                          660  700
 |    |                            |    |
 █████                              █████  ← AI 천장 없음 (aiDeathZone y=10 센서)
 █  [aiRightFlipper]  [aiLeftFlipper]   █  ← y=105 (AI, 빨간색)
 █      \                    /         █
 █    aiRightFunnel    aiLeftFunnel     █    바깥끝 x=40/660, 안끝 x=254.5/445.5
 █                                     █
 █         [범퍼]   [목표물]            █
 █                                     █
 █    [leftFlipper]  [rightFlipper]    █  ← y=995 (플레이어, 주황색)
 █      /                    \         █    고정축 x=242.5, x=457.5
 █  leftFunnel         rightFunnel     █    바깥끝 x=40/660, 안끝 x=254.5/445.5
 ████  [    deathZone y=1090 센서    ]  ████
```

### AI 판단 로직

`beforeUpdate` 이벤트마다 실행:
- 조건: `ball.y < 300` AND `ball.vy < 0` (위로 올라가는 중)
- AI 플리퍼 중심 X 기준 160px 이내면 해당 플리퍼 올림
- 조건 불충족 시 플리퍼 내림 (기본 상태)

### 플레이스타일 분석 흐름

```
게임 시작
  └─ 100ms마다 수집 (공이 flipperTrigger 센서 내부일 때만)
       수집: timestamp, ball_x/y, ball_vx/vy, left_flipper, right_flipper

10초 후
  └─ POST /api/v1/pinball_ai/playstyle
       attack  → skillState = 'small' (작은 공)
       defence → skillState = 'big'   (큰 공)
       none    → 스킬 없음
       30초 무응답 → 랜덤 폴백
```

### 필살기 시스템

| 스킬 | 공 반지름 | 효과 | 지속 |
|------|-----------|------|------|
| `big` | r=30 (기본 2배) | 무겁고 범퍼를 강하게 밀어냄 | 5초 |
| `small` | r=10.5 (기본 0.7배) | 가볍고 빠름 | 5초 |

발동: 스킬 아이콘 클릭 → 현재 공 교체 → 5초 후 자동 원복

### 주요 상태값

| 상태 | 설명 |
|------|------|
| `skillState` / `skillStateRef` | `'loading'` \| `'big'` \| `'small'` |
| `cooldownProgress` | 스킬 아이콘 충전 진행도 (0~1, 10초 기준) |
| `specialActiveRef` | 필살기 발동 중 여부 (중복 방지) |
| `isBallInTriggerRef` | 공이 flipperTrigger 센서 내부 여부 |
| `playstyleDataRef` | 수집된 플레이스타일 데이터 배열 |

## 현재 게임 플레이 시나리오

### 1단계 — 게임 진입 (대기 화면)

- 화면: 검정 반투명 오버레이 위에 **"AI PINBALL"** 타이틀
- 조작 안내 표시 (PC):
  - `← →` : 플립퍼 작동
  - "AI 플리퍼가 상단에서 공을 막습니다"
- 물리 엔진은 `timeScale = 0` 으로 정지된 상태
- **공은 필드 중앙 `(x=350, y=530)` 에 배치됨**
- PC: `Space` 키 또는 화면 클릭 → 게임 시작
- 모바일: 화면 터치 → 게임 시작

---

### 2단계 — 게임 시작

`startGame()` 호출 시:

1. `timeScale = 1` → 물리 엔진 활성화, BGM 재생
2. **플레이스타일 수집 시작** (`collectInterval`, 100ms 주기)
   - 공이 `flipperTrigger` 센서(플리퍼 영역) 안에 있을 때만 기록
   - 수집 항목: `timestamp`, `ball_x/y`, `ball_vx/vy`, `left_flipper`, `right_flipper`
3. **스킬 아이콘 충전 시작** (`cooldownInterval`, 10초 기준 진행도 표시)
4. **10초 후 자동 실행** (`analysisTimer`):
   - 수집 종료 → `POST /api/v1/pinball_ai/playstyle` 호출
   - `attack` 응답 → `skillState = 'small'`
   - `defence` 응답 → `skillState = 'big'`
   - `none` 응답 → 스킬 없음
   - 30초 내 무응답 → `getRandomSkill()` 랜덤 폴백

---

### 3단계 — 인게임 플레이

#### 플레이어 조작

| 입력 | 동작 |
|------|------|
| `←` (누르는 동안) | 왼쪽 플리퍼 올림 (-35°) |
| `→` (누르는 동안) | 오른쪽 플리퍼 올림 (+35°) |
| `A` 키 | 필살기 발동 (스킬 상태가 `big`/`small`일 때만) |
| `N` 키 | 다음 스테이지로 강제 전환 (테스트용) |
| 모바일 좌측 터치 | 왼쪽 플리퍼 |
| 모바일 우측 터치 | 오른쪽 플리퍼 |

#### AI 플리퍼 동작 (자동)

매 물리 프레임(`beforeUpdate`)마다 판단:
- 조건: `ball.y < 300` AND `ball.vy < 0` (공이 상단으로 올라오는 중)
- AI 플리퍼 중심 X 기준 160px 이내 → 해당 플리퍼 올림
- 조건 미충족 → 플리퍼 내림 (기본 대기 상태)

#### 충돌 이벤트

| 충돌 대상 | 결과 |
|-----------|------|
| 범퍼 | 공에 반발 속도 추가(+15), 범퍼 사운드 |
| 목표물(target) | +300점, 점수 팝 애니메이션 |
| deathZone | 목숨 -1, 목숨 남으면 공을 중앙(x=350, y=550)에서 `launchDirectionRef` 방향으로 재발사 |
| deathZone (목숨 0) | 공 제거, 게임오버 overlay 표시 |

---

### 4단계 — 필살기 발동 (스킬 획득 후)

1. 게임 시작 10초 후 스킬 아이콘에 `BIG` 또는 `SMALL` 표시
2. `A` 키 (또는 스킬 아이콘 클릭) 으로 발동
3. 현재 공 World에서 제거 → 특수 공을 같은 위치/속도로 투입
4. 5초 후 자동으로 원래 공으로 교체

| 스킬 | 공 크기 | 특징 |
|------|---------|------|
| `big` | r=30 (2배) | 무거움, 범퍼 충돌 시 강하게 튕김 |
| `small` | r=10.5 (0.7배) | 가벼움, 빠르고 좁은 틈 통과 |

---

### 5단계 — 게임오버

- overlay: "GAME OVER" + 획득 점수 표시
- 재시작 버튼 → `handleRestart()`:
  - score/lives/stage 초기화 (0, 3, 1)
  - 공을 중앙 `(x=350, y=550)` 에서 `launchDirectionRef` 방향으로 재발사
  - 스테이지 1 맵 재로딩
  - ※ Pinball과 달리 점수 API 제출 없음, 세션 저장/삭제 없음

---

### 현재 미구현 / 미완성 항목

- 스킬 아이콘 클릭으로 필살기 발동하는 UI 핸들러 없음 (`A` 키만 동작)
- 게임오버 후 `skillState`가 초기화되지 않음 (재시작해도 이전 스킬 유지)
- 스테이지 클리어 조건 없음 (`targetScore` 정의되어 있으나 체크 로직 없음)
- 플레이스타일 수집이 재시작 후 재실행되지 않음 (재시작 시 분석 타이머 미복구)

## 개편 계획

### AI 라이프 추가

**목적**: 상대(AI)에게도 라이프를 부여해서 플레이어 vs AI 대결 구조 완성

**변경 내용**:

1. **상태 추가** (`AIPinball.jsx`)
   - `aiLives` useState (초기값 3)
   - `aiLivesRef` useRef (콜백 내 최신값 읽기용)

2. **AI deathZone 추가** (Matter.js)
   - 천장(`upWall`) 위쪽에 센서 배치 (`y ≈ -30`)
   - 공이 여기 충돌하면 AI 목숨 -1
   - AI 목숨 > 0: 공을 중앙으로 복귀
   - AI 목숨 == 0: 플레이어 승리 overlay 표시

3. **UI 상단바 수정**
   - 현재: 플레이어 하트만 표시
   - 변경: AI 하트(다른 색)를 우측에 추가

4. **재시작 핸들러 수정**
   - `aiLives` / `aiLivesRef` 도 3으로 초기화

5. **승리 overlay 추가**
   - `overlayState = 'playerWin'` 케이스 추가
   - "YOU WIN" 텍스트 + 재시작 버튼

**변경 파일**: `AIPinball.jsx`

### 플런저 제거 + 공 중앙 출현

**목적**: AIPinball은 대결 구조이므로 플런저 발사 방식 대신 공이 필드 중앙에서 자동 시작

**변경 내용**:

1. **제거 대상** (`AIPinball.jsx`)
   - `plunger`, `plungerShelf`, `plungerLaneGuide` 물리 바디 삭제
   - `plungerRef`, `plungerStartRef`, `plungerReleaseRef` ref 삭제
   - `isSpacePressedRef`, `spaceHoldStartTimeRef` 삭제
   - `beforeUpdate` 내 Plunger 당기기 로직 삭제
   - Space 키 플런저 충전/발사 핸들러 삭제
   - 모바일 플런저 버튼 UI 삭제
   - `rightWall2` (Plunger lane 내벽) 삭제

2. **공 시작 방식 변경**
   - 공 위치: 필드 중앙 `(x=350, y=550)`
   - 발사 방향: 왼쪽 아래 또는 오른쪽 아래 랜덤 (`Math.random() < 0.5`)
     - 왼쪽 아래: `vx = -속도, vy = +속도`
     - 오른쪽 아래: `vx = +속도, vy = +속도`
   - 게임 시작 시 랜덤 방향 결정, 목숨 감소 후 복귀 시도 동일한 랜덤 방향으로 발사

3. **Space 키 용도 변경 또는 제거**
   - Space로 게임 시작하는 부분은 유지
   - 게임 중 Space 플런저 로직만 제거

**변경 파일**: `AIPinball.jsx`

### 맵 구성 변경

**목적**: Plunger lane 공간을 제거하고 전체 폭을 플레이 구역으로 확장, 좌우 대칭 구조로 개편

**현재 맵**:
```
x=0  40      630 660  700
 |    |        |   |    |
 ████████████████████████  ← upWall (y=0~40)
 █    |        |   |    █
 █    |        |   ██████  ← rightWall2 (Plunger lane 내벽)
 █  [aiR]  [aiL]  |    █  ← AI 플리퍼 (y=105)
 █    \      /    |    █
 █     범퍼/목표물  |    █
 █                |    █
 █  [L플리퍼][R플리퍼]  █  ← 플레이어 플리퍼 (y=995)
 █   /          \      █
 ████  deathZone  ████     ← 깔대기+deathZone (y=1195)
```

**변경 후 맵**:
```
x=0  40    200       500  660  700
 |    |     |         |    |    |
 ████████   [aiDeathZone]  ████████  ← 천장 좌우 벽 + 가운데 구멍
 █  [aiR플리퍼]  [aiL플리퍼]       █  ← AI 플리퍼 (y=105)
 █      \            /            █
 █       aiFunnel 경사             █
 █                                 █
 █       범퍼 / 목표물              █
 █                                 █
 █     playerFunnel 경사            █
 █      /            \             █
 █  [L플리퍼]  [R플리퍼]            █  ← 플레이어 플리퍼 (y=995)
 ████   [  deathZone 센서  ]   ██████  ← 바닥 좌우 벽 + 가운데 구멍
```

- 플레이어 deathZone: 바닥 깔대기 구멍 아래 센서 (기존 방식 유지)
- AI deathZone: 천장을 좌우 벽 2개로 분리하고 가운데 구멍을 뚫어 그 위에 센서 배치
  - 좌측 천장 벽: `x=0 ~ x=200`
  - 우측 천장 벽: `x=500 ~ x=700`
  - 가운데 구멍: `x=200 ~ x=500` (AI 깔대기가 공을 이 구멍으로 유도)
  - aiDeathZone 센서: 구멍 위 `y ≈ -30`

**변경 내용**:
1. `rightWall2` 제거 (Plunger lane 내벽 삭제)
2. `plungerLaneGuide` 제거
3. `upWall` (단일 천장) → 좌측 천장 + 우측 천장 2개로 분리 (가운데 구멍)
4. AI 깔대기(`aiFunnelWall`) 가 공을 천장 구멍으로 유도
5. aiDeathZone 센서 추가 (천장 구멍 위)
6. 전체 플레이 구역 좌우 대칭 확보

**변경 파일**: `AIPinball.jsx`
