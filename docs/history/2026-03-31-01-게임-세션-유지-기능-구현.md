# 게임 세션 유지 기능 구현

**날짜**: 2026-03-31  
**상태**: 구현 예정

---

## 작업 개요

핀볼 게임 도중 페이지를 나갔다가 돌아와도 이전 게임 상태(점수·목숨·스테이지)를 이어서 진행할 수 있도록 세션 유지 기능을 추가한다.

**적용 대상**: 로그인 사용자만  
**저장 대상**: 점수, 목숨, 스테이지 (공 위치 제외 — 복귀 시 스테이지 시작 위치에서 재개)  
**저장 위치**: Redis (주) + localStorage (fallback)

---

## 주요 결정 사항

| 결정 | 선택 | 이유 |
|------|------|------|
| 적용 대상 | 로그인 사용자만 | 비로그인 사용자는 서버 저장 불가 |
| 저장 위치 | Redis + localStorage | 임시 데이터에 적합, TTL로 자동 만료 + 서버 오류 시 fallback |
| 복구 정밀도 | 점수·목숨·스테이지만 | 공 위치는 물리 재현이 복잡하고 체감 차이 작음 |
| 세션 수 | 사용자당 1개 (UNIQUE 제약) | 진행 중 세션은 항상 최신 1개만 유지 |
| 저장 타이밍 | 30초마다 + 목숨 감소 시 즉시 | 목숨 손실은 가장 중요한 상태 변화 |

---

## 구현 범위

### 백엔드

- `docker-compose.yml` — Redis 서비스 추가
- `backend/requirements.txt` — `redis` 패키지 추가
- `backend/app/redis_client.py` — Redis 연결 클라이언트 신규 생성
- `backend/app/routers/game_sessions.py` — 신규 생성
  - `PUT /api/v1/game-sessions/{user_id}` — Redis에 세션 저장 (upsert, TTL 설정)
  - `GET /api/v1/game-sessions/{user_id}` — Redis에서 세션 조회 (없으면 null)
  - `DELETE /api/v1/game-sessions/{user_id}` — Redis에서 세션 삭제
- `backend/app/main.py` — 라우터 등록
- Alembic 마이그레이션 불필요 (Redis는 스키마 없음)

### 프론트엔드

- `frontend/src/utils/gameSession.js` — 신규 생성
  - `saveGameSession`, `loadGameSession`, `deleteGameSession` (서버)
  - `saveSessionLocal`, `loadSessionLocal`, `clearSessionLocal` (localStorage)
- `frontend/src/pages/Pinball.jsx` — 수정
  - 마운트 시 세션 복원 로직 추가
  - 30초마다 + 목숨 감소 시 저장
  - 게임 오버 / 재시작 시 세션 삭제

---

## 저장 흐름

```
[페이지 진입]
  로그인 유저 → GET /api/v1/game-sessions/{user_id}
    세션 있음 → 점수/목숨/스테이지 복원, 공은 시작 위치
    세션 없음 → 초기값(0, 3, 1)으로 새 게임

[게임 진행 중]
  30초마다 + 목숨 감소 시 → PUT /api/v1/game-sessions/{user_id} → Redis 저장 + localStorage

[게임 오버]
  DELETE /api/v1/game-sessions/{user_id} + localStorage 삭제
  점수 제출 (기존 submitScore 유지)
```
