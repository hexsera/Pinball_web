# 게임 세션 유지 기능 구현

**날짜**: 2026-03-31  
**상태**: 진행 중 (Redis 설치 완료 / API·프론트엔드 미구현)  
**연관 문서**: [2026-03-31-02-Redis-설치.md](2026-03-31-02-Redis-설치.md)

---

## 작업 개요

핀볼 게임 도중 페이지를 나갔다가 돌아와도 이전 게임 상태(점수·목숨·스테이지)를 이어서 진행할 수 있도록 세션 유지 기능을 추가한다.

**적용 대상**: 로그인 사용자만  
**저장 대상**: 점수, 목숨, 스테이지 (공 위치 제외 — 복귀 시 스테이지 시작 위치에서 재개)  
**저장 위치**: Redis (주) + localStorage (fallback)

---

## 주요 결정 사항

| 결정 | 선택 | 이유 |
|------|------|------|
| 적용 대상 | 로그인 사용자만 | 비로그인 사용자는 서버 저장 불가 |
| 저장 위치 | Redis + localStorage | 임시 데이터에 적합, TTL로 자동 만료 + 서버 오류 시 fallback |
| 복구 정밀도 | 점수·목숨·스테이지만 | 공 위치는 물리 재현이 복잡하고 체감 차이 작음 |
| 세션 수 | 사용자당 1개 (Redis key로 구분) | 진행 중 세션은 항상 최신 1개만 유지 |
| 저장 타이밍 | 30초마다 + 목숨 감소 시 즉시 | 목숨 손실은 가장 중요한 상태 변화 |

```python
# backend/app/redis_client.py
import redis, os

redis_client = redis.Redis(
    host=os.getenv("REDIS_HOST", "redis-server"),
    port=int(os.getenv("REDIS_PORT", 6379)),
    decode_responses=True
)
```

- `backend/.env.example`에 `REDIS_HOST`, `REDIS_PORT` 항목 추가

## 미완료 작업

### 1. 백엔드: game_sessions 라우터 구현

**생성할 파일**: `backend/app/api/v1/game_sessions.py`

기존 라우터 파일들(`monthly_scores.py`, `game_visits.py`) 패턴을 그대로 따른다.

```
PUT    /api/v1/game-sessions/{user_id}
       요청: { score: int, lives: int, stage: int }
       응답: { user_id, score, lives, stage, updated_at }
       설명: 세션 저장 (없으면 생성, 있으면 덮어씀). TTL 24시간 설정.

GET    /api/v1/game-sessions/{user_id}
       응답: { user_id, score, lives, stage, updated_at } 또는 null
       설명: 저장된 세션 조회. 없으면 null 반환 (404 아님).

DELETE /api/v1/game-sessions/{user_id}
       응답: { message: "session deleted" }
       설명: 게임 오버 또는 재시작 시 세션 삭제.
```

**Redis key 구조**: `game_session:{user_id}` (JSON 문자열로 저장)

**참고할 파일**:
- 라우터 패턴: `backend/app/api/v1/monthly_scores.py`
- Redis 클라이언트: `backend/app/redis_client.py`
- 스키마 패턴: `backend/app/schemas/monthly_score.py`

**스키마 추가 위치**: `backend/app/schemas/game_session.py` 신규 생성 후 `backend/app/schemas/__init__.py`에 export 추가

**라우터 등록 위치**: `backend/app/main.py` (기존 라우터 등록 방식 확인 후 동일하게 추가)

### 2. 프론트엔드: 세션 유틸리티 생성

**생성할 파일**: `frontend/src/utils/gameSession.js`

```javascript
// 서버(Redis) 연동
export async function saveGameSession(userId, { score, lives, stage })
export async function loadGameSession(userId)   // 없으면 null 반환
export async function deleteGameSession(userId)

// localStorage fallback
export function saveSessionLocal(userId, session)
export function loadSessionLocal(userId)        // 없으면 null 반환
export function clearSessionLocal(userId)
```

localStorage key: `pinball_session_{userId}`

### 3. 프론트엔드: Pinball.jsx 수정

**파일**: `frontend/src/pages/Pinball/Pinball.jsx`

현재 상태값 (수정 대상):
- `score` / `scoreRef` (line 48, 37)
- `lives` / `livesRef` (line 50, 36)
- `stage` / `stageRef` (line 51, 38)
- `handleRestart()` (line 111): 현재 항상 초기값(0, 3, 1)으로 리셋
- `submitScore()` (line 91): 게임 오버 시 점수 서버 전송 — 이후에 세션 삭제 추가

**추가할 로직**:

1. **마운트 시 세션 복원** (새 `useEffect` 추가, 기존 game_visits useEffect 근처):
   - 로그인 사용자라면 `loadGameSession(user.id)` 호출
   - 세션 있으면 score, lives, stage 및 해당 ref 값을 복원
   - 없으면 기존대로 초기값 유지

2. **주기적 저장** (새 `useEffect` 추가):
   - 30초마다 `saveGameSession` + `saveSessionLocal` 호출
   - 컴포넌트 언마운트 시 interval 정리

3. **목숨 감소 시 즉시 저장**:
   - deathZone 충돌 감지 코드에서 `livesRef.current` 변경 직후 `saveGameSession` 호출

4. **게임 오버 시 세션 삭제**:
   - `submitScore()` 호출 이후 `deleteGameSession` + `clearSessionLocal` 호출

5. **handleRestart() 수정**:
   - 초기화 로직 이후 `deleteGameSession` + `clearSessionLocal` 호출

## 저장 흐름 요약

```
[페이지 진입]
  로그인 유저 → GET /api/v1/game-sessions/{user_id}
    세션 있음 → score/lives/stage 복원, 공은 시작 위치
    세션 없음 → 초기값(0, 3, 1)으로 새 게임

[게임 진행 중]
  30초마다 + 목숨 감소 시 → PUT /api/v1/game-sessions/{user_id} + localStorage

[게임 오버 / 재시작]
  DELETE /api/v1/game-sessions/{user_id} + localStorage 삭제
  점수 제출 (기존 submitScore 유지)
```
