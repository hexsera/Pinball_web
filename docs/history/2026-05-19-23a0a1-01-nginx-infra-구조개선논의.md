# Nginx 설정 파일 위치 및 infra 디렉토리 구조 개선 논의

## 현상 파악

`docker-compose.yml`의 nginx 볼륨 마운트 확인:

```yaml
volumes:
  - ./nginx.conf:/etc/nginx/nginx.conf:ro
```

루트의 `./nginx.conf`를 사용 중. `./nginx/nginx.conf`는 존재하지만 **실제로는 무시되는** 상태.

두 파일 비교:

| | `./nginx.conf` (실제 사용) | `./nginx/nginx.conf` (미사용) |
|---|---|---|
| 포트 | 80 | 8080 |
| root | `/etc/nginx/html` | `/home/hexsera/long_time_test/...` |
| mime.types | `include /etc/nginx/mime.types` | 직접 types 블록 정의 |

`./nginx/nginx.conf`는 예전 로컬 테스트용 파일로 판단.

## 논의 결론

### nginx 설정 파일의 표준 위치

프로그래밍 관례상 설정 파일은 프로젝트 루트가 아닌 **전용 디렉토리** 안에 위치해야 한다.

현재 프로젝트는 traefik.yml도 루트에 있어 인프라 설정 파일들이 루트를 오염시키는 구조.

### 채택할 디렉토리 구조

```
project/
├── backend/
├── frontend/
├── infra/
│   ├── nginx/
│   │   └── nginx.conf
│   └── traefik/
│       └── traefik.yml
└── docker-compose.yml
```

- `infra/` 하위에 nginx, traefik 설정을 모아 관리
- `docker-compose.yml`은 관례상 루트에 유지
- 볼륨 경로를 `./infra/nginx/nginx.conf`로 변경
