# 게임 세션 API 백엔드 구현 실행계획

## 요구사항 요약

**요구사항**: Redis를 저장소로 사용하는 게임 세션 REST API(GET/PUT/DELETE)를 백엔드에 구현한다.

**목적**: 프론트엔드(`gameSession.js`)가 호출하는 `/api/v1/game-sessions/{user_id}` 엔드포인트가 없어 세션 저장·복원·삭제가 동작하지 않는다. 백엔드를 완성해 전체 기능을 연결한다.

---

## 현재상태 분석

- 프론트엔드: `gameSession.js` 구현 완료, `Pinball.jsx` 수정 완료
- Redis 인프라: `docker-compose.yml`에 redis 서비스 추가 완료, `backend/app/redis_client.py` 생성 완료
- 백엔드: `game_sessions.py` 라우터 없음, 스키마 없음, `main.py`에 라우터 미등록

---

## 구현 방법

- Pydantic으로 요청/응답 스키마를 정의하고, FastAPI 라우터에서 `app/redis_client.py`의 `redis_client`를 import해서 JSON 직렬화로 Redis에 저장한다.
- DB(PostgreSQL)는 사용하지 않는다. 세션은 임시 데이터이므로 Redis TTL(7200초)로 자동 만료시킨다.
- 기존 라우터(`monthly_scores.py`)의 파일 구조·import 패턴을 따른다.

---

## 구현 단계

### 1. 스키마 파일 생성 (`backend/app/schemas/game_session.py`)

```python
from pydantic import BaseModel

class GameSessionSaveRequest(BaseModel):
    score: int
    lives: int
    stage: int

class GameSessionResponse(BaseModel):
    user_id: int
    score: int
    lives: int
    stage: int
    updated_at: str

class GameSessionDeleteResponse(BaseModel):
    message: str
```

- **무엇을 하는가**: PUT 요청 바디, GET/PUT 응답, DELETE 응답을 Pydantic 타입으로 정의
- `updated_at`은 Redis에 ISO 형식 문자열로 저장하므로 `str` 타입 사용
- ORM 모델이 아닌 dict에서 직접 생성하므로 `Config(from_attributes=True)` 불필요

---

### 3. 라우터 파일 생성 (`backend/app/api/v1/game_sessions.py`)

```python
import json
from datetime import datetime, timezone
from fastapi import APIRouter, HTTPException
from app.redis_client import redis_client
from app.schemas.game_session import (
    GameSessionSaveRequest, GameSessionResponse, GameSessionDeleteResponse
)

REDIS_TTL = 7200
router = APIRouter()

@router.put("/{user_id}", response_model=GameSessionResponse)
def save_game_session(user_id: int, data: GameSessionSaveRequest):
    key = f"game_session:{user_id}"
    payload = {**data.model_dump(), "user_id": user_id,
               "updated_at": datetime.now(timezone.utc).isoformat()}
    redis_client.set(key, json.dumps(payload), ex=REDIS_TTL)
    return payload

@router.get("/{user_id}", response_model=GameSessionResponse)
def get_game_session(user_id: int):
    key = f"game_session:{user_id}"
    raw = redis_client.get(key)
    if raw is None:
        raise HTTPException(status_code=404, detail="Session not found")
    return json.loads(raw)

@router.delete("/{user_id}", response_model=GameSessionDeleteResponse)
def delete_game_session(user_id: int):
    key = f"game_session:{user_id}"
    redis_client.delete(key)
    return GameSessionDeleteResponse(message="session deleted")
```

- **무엇을 하는가**: Redis key `game_session:{user_id}`에 JSON으로 세션을 저장·조회·삭제
- PUT은 없으면 생성, 있으면 덮어씀 (`set`은 upsert 동작)
- DELETE는 세션 없어도 에러 없이 성공 응답 (멱등성 보장)
- `decode_responses=True`로 설정된 redis_client는 `get()` 결과가 str이므로 `json.loads` 바로 사용 가능
