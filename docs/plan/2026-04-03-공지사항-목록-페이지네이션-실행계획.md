# 공지사항 목록 페이지네이션 실행계획

## 요구사항 요약

**요구사항**: 공지사항 목록 페이지에서 전체 공지를 한 번에 보여주는 대신, 10개씩 나눠서 페이지 번호 버튼으로 이동하도록 변경한다.

**목적**: 공지사항이 많아질수록 한 페이지에 모두 표시하면 스크롤이 길어지고 로딩이 느려진다. 10개씩 나눠 보여주면 사용자가 원하는 페이지로 바로 이동할 수 있고 초기 로딩 데이터도 줄어든다.

## 현재상태 분석

- `NoticeListPage.jsx`: `getNotices()`를 인자 없이 호출해 전체 목록을 배열로 받아 그대로 렌더링
- `noticeService.js`: `getNotices()`가 `GET /api/v1/notices`를 파라미터 없이 호출, 실패 시 IndexedDB 전체 목록 반환
- `noticeService.test.js`: `getNotices()`가 파라미터 없이 배열을 반환하는 것을 검증 → 수정 시 깨짐
- `NoticeListPage.test.jsx`: `getNotices` mock이 배열을 반환하도록 설정 → 수정 시 깨짐
- 페이지 상태 관리 없음, URL 쿼리스트링 미사용

## 구현 방법

- `useSearchParams`로 URL의 `?page=1` 쿼리스트링을 읽어 현재 페이지를 관리한다. 뒤로가기/새로고침 시 페이지가 유지된다.
- `getNotices(skip, limit)`에 파라미터를 추가해 `?skip=0&limit=10` 형태로 API를 호출한다. 응답은 `{ items, total }` 구조다.
- `total`과 `limit(=10)`으로 전체 페이지 수를 계산하고, 현재 페이지 기준으로 5개씩 슬라이딩하는 페이지 번호 버튼을 렌더링한다.
- IndexedDB fallback도 동일한 `{ items, total }` 구조로 반환하도록 수정해 일관성을 유지한다.
- TDD 방식으로 진행한다: 테스트 수정/추가 → 구현 순서로 작업한다.

- 응답 구조가 배열에서 `{ items, total }` 객체로 바뀐 것을 검증한다.

### 2. noticeService.js 수정

```javascript
export const getNotices = async (skip = 0, limit = 10) => {
  try {
    const res = await axios.get('/api/v1/notices', { params: { skip, limit } });
    return res.data; // { items, total }
  } catch {
    await seedNoticesIfEmpty();
    const all = await getAllNoticesFromDB();
    return {
      items: all.slice(skip, skip + limit),
      total: all.length,
    };
  }
};
```
- `skip`, `limit` 파라미터를 받아 API에 전달한다. 기본값은 각각 `0`, `10`.
- IndexedDB fallback도 `{ items, total }` 구조로 반환해 API 성공/실패 시 응답 형태를 통일한다.
- `all.slice(skip, skip + limit)`으로 해당 페이지 데이터만 잘라 반환한다.

### 4. NoticeListPage.jsx 수정

```javascript
const [searchParams, setSearchParams] = useSearchParams();
const page = parseInt(searchParams.get('page') || '1', 10);
const LIMIT = 10;

const [notices, setNotices] = useState([]);
const [total, setTotal] = useState(0);

useEffect(() => {
  const skip = (page - 1) * LIMIT;
  getNotices(skip, LIMIT).then(({ items, total }) => {
    setNotices(items);
    setTotal(total);
  });
}, [page]);

const totalPages = Math.ceil(total / LIMIT);
// 5개씩 슬라이딩: 현재 페이지 기준으로 표시할 페이지 번호 범위 계산
const pageGroupStart = Math.floor((page - 1) / 5) * 5 + 1;
const pageGroupEnd = Math.min(pageGroupStart + 4, totalPages);
```
- `useSearchParams`로 URL `?page=N`을 읽어 현재 페이지를 결정한다.
- `page` 변경 시 `useEffect`가 재실행되어 해당 페이지 데이터를 다시 불러온다.
- `pageGroupStart ~ pageGroupEnd` 범위의 버튼만 렌더링해 최대 5개씩 표시한다.

### 5. 페이지 버튼 UI 렌더링

```jsx
<Box sx={{ display: 'flex', justifyContent: 'center', gap: 1, mt: 2 }}>
  <Button disabled={page === 1} onClick={() => setSearchParams({ page: page - 1 })}>
    &lt;
  </Button>
  {Array.from({ length: pageGroupEnd - pageGroupStart + 1 }, (_, i) => {
    const p = pageGroupStart + i;
    return (
      <Button key={p} variant={p === page ? 'contained' : 'outlined'}
              onClick={() => setSearchParams({ page: p })}>
        {p}
      </Button>
    );
  })}
  <Button disabled={page === totalPages} onClick={() => setSearchParams({ page: page + 1 })}>
    &gt;
  </Button>
</Box>
```
- `<` `>` 버튼으로 이전/다음 페이지로 이동한다. 첫/마지막 페이지에서는 비활성화된다.
- 현재 페이지 버튼은 `variant="contained"`, 나머지는 `variant="outlined"`로 구분한다.
- `setSearchParams({ page: p })`로 URL을 변경하면 `useEffect`가 트리거되어 데이터를 다시 불러온다.
